# Track C1: Zstd Fuel Usage Audit

Session: 465f2539 | Date: 2026-03-13

## Overview

This audit catalogs all fuel-based recursion in the Zstd spec and native
layers, identifies decreasing measures for well-founded recursion, and
proposes a conversion plan.

**Key finding**: Fuel usage in Zstd is concentrated in exactly two functions
in `Zip/Native/Fse.lean`. All other Zstd native and spec files already use
well-founded recursion or structural recursion. The Zstd fuel elimination
scope is much smaller than the DEFLATE side.

## Conversion Plan

### Issue 1: Convert `decodeZeroRepeats` to WF recursion

**Scope**: `Zip/Native/Fse.lean` + 4 theorems in `Zip/Spec/Fse.lean`
**Measure**: `br.data.size * 8 - br.bitPos`
**Prerequisites**: Need `readBits_bitPos_strict_lt` (readBits with n > 0 strictly
advances bitPos). The existing `readBits_bitPos_eq` likely already proves this.
**Steps**:
1. Refactor `decodeZeroRepeats` to take `(fuel := br.data.size * 8 - br.bitPos)`
   or rewrite as a `def ... termination_by br.data.size * 8 - br.bitPos`
2. Prove `decreasing_by` using existing bitPos advancement lemmas
3. Update 4 dependent theorems to use `.induct` instead of `induction fuel`
**Difficulty**: 3/10

### Issue 2: Convert `decodeFseLoop` to WF recursion

**Scope**: `Zip/Native/Fse.lean` + 4 theorems in `Zip/Spec/Fse.lean`
**Measure**: `(maxSymbols - symbolNum) + remaining`
**Prerequisites**: Need a lemma `decodeZeroRepeats_symbolNum_ge` that the returned
`symbolNum ‚â• input symbolNum`. The existing `decodeZeroRepeats_cellCount` likely
provides enough information.
**Steps**:
1. Prove `decodeZeroRepeats_symbolNum_ge` helper lemma
2. Refactor `decodeFseLoop` to use `termination_by (maxSymbols - symbolNum) + remaining`
3. Prove `decreasing_by` for all three branches
4. Update 4 dependent theorems to use `.induct` instead of `induction fuel`
5. Update 3 higher-level theorems that transitively depend on them
**Difficulty**: 5/10

### Recommended Order

Convert `decodeZeroRepeats` first (Issue 1) ‚Äî it's a dependency of `decodeFseLoop`,
and converting it first ensures the WF version is available when working on Issue 2.
