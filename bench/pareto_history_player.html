Compression speed vs ratio — the history

The compress Pareto with the reference curves fixed at the latest run, and the native lean-zip curve replayed through every dashboard refresh (, frames). Top-left is ideal.

Reading it: each marker is a compression level; x = geomean compressed/original over the corpus files (smaller is better), y = geomean compress MB/s (log). Connectors are the achievable mixing frontier between adjacent levels, not straight segments. Native endpoints are tagged with their level; trails show the path each level has traced so far.

Provenance: native curve replayed from the git history of bench/results/latest.json; reference curves from the current data @ (, ). zopfli is the frozen ratio ceiling (single-rep speed, indicative only). Throughput is a median snapshot; ratios are deterministic.
