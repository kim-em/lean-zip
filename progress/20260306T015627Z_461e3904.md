## Proof technique

**FSE proofs**: Unfold `decodeFseDistribution`, case-split on `readBits` result
via `cases`, then peel through 5 layers of `if`/`match` using `split at h`
(dismissing all `.error` branches with `exact nomatch h`). Extract `al = val.fst.toNat + 5`
via `Except.ok.injEq` + `Prod.mk.injEq`, close with `omega`.

**Frame header proof**: Same pattern but `parseFrameHeader` is too large for `split`
(hits simp step limit). Used `by_cases` + `rw [if_neg/if_pos]` instead, which avoids
simp's internal traversal. Extracted magic equality via Bool case analysis on `==`.

## Key patterns discovered

- For large unfolded functions, `by_cases` + `rw [if_neg/if_pos]` is more scalable
  than `split at h` (which uses simp internally and can hit step limits).
- `simp only [pure, Pure.pure]` reduces `match pure PUnit.unit with ...` patterns
  that `dsimp only` cannot handle (because `pure` goes through typeclass instances).
- `eq_of_beq` converts `(a == b) = true` to `a = b` for types with `LawfulBEq`.
