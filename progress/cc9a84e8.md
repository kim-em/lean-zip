## What was done

Converted all `]!` (runtime-checked) array accesses in the recursive
`lz77Greedy` and its `where` helpers (`hash3`, `countMatch`, `go`,
`trailing`, `updateHashes`, `mainLoop`) to proven-bounds `]'h` form.
This mirrors the iterative counterpart `lz77GreedyIter` (cluster B,
PR #1517) so both greedy matchers now use the same proven-bounds
pattern.

### `Zip/Native/Deflate.lean`
- Added bound hypotheses to helper signatures:
  - `hash3` takes `(h : pos + 2 < data.size)`
  - `countMatch`/`go` take `(h1 : p1 + maxLen ≤ data.size)` and
    `(h2 : p2 + maxLen ≤ data.size)`
- `mainLoop` gained explicit `dite` guards on `pos + 2 < data.size`,
  `h < hashTable.size`, `h < hashValid.size`, and the match condition
  is now a single `∧`-proposition (`isValid ∧ matchPos < pos ∧
  pos - matchPos ≤ windowSize`) rather than a `&&` Boolean expression.
- `trailing` switched to `if h : pos < data.size` with proven-bounds
  literal emission.

### `Zip/Spec/LZ77NativeCorrect.lean`
- Threaded `h1`/`h2` parameters through `lz77Greedy.go_matches` and
  `lz77Greedy.countMatch_matches`.
- Refactored `lz77Greedy.mainLoop_valid`, `mainLoop_encodable`, and
  `mainLoop_length` to match the six-level `dite` structure (hlt, hht,
  hhv, hcond, hge, hle) and use the `∧`-shaped `hcond` destructuring.
- `trailing_valid` updated to use proven-bounds `data[pos]'hlt`.

### `Zip/Spec/DeflateFixedCorrect.lean`
- `lz77GreedyIter` and `lz77Greedy` now share identical helper bodies
  for `hash3`, `go`, `countMatch`, `updateHashes`. The
  `*_eq` lemmas simplify substantially:
  - `hash3_eq` is `rfl`.
  - Others drop all `getElem!_pos` bridging rewrites.
- `mainLoop_eq` dropped the `hand`/`Bool.and_eq_true`/`decide`
  conversions and the `getElem!_pos data pos …` rewrites — the
  `dite`-structures on both sides match directly. `dif_pos`/`dif_neg`
  replace the old `&&`-based `ite` reductions.

### `Zip/Spec/DeflateDynamicCorrect.lean`
- One local rewrite switched from `if_neg` to `dif_neg` for the empty-
  input branch of `lz77Greedy.trailing`.

## Technical notes

- Greedy and GreedyIter helpers are now structurally identical modulo
  accumulator vs. list-cons, which makes the translation-validation
  proofs short.
- The dead-branch pattern (emit a literal when `h < hashTable.size` /
  `hashValid.size` fails) is extensionally equivalent to the original
  `!`-default behaviour because `replicate hashSize 0/false` means
  `matchPos` would have been a literal 0 and `isValid` `false`, both
  of which would have fallen through the match condition.
- No statement weakening in spec cascade — only proof bodies were
  adjusted; public theorem signatures are unchanged.
