# Progress: Frame-level content for raw/RLE+compSeq two-block frames

- **Date**: 2026-03-10T08:54Z
- **Session type**: feature
- **Issue**: #1058

## Accomplished

Added two frame-level content theorems to `Zip/Spec/Zstd.lean`:

1. **`decompressFrame_raw_then_compressed_seq_content`** — When a frame contains
   a non-last raw block followed by a last compressed block with numSeq>0, the
   output equals `block1 ++ blockOutput2`.

2. **`decompressFrame_rle_then_compressed_seq_content`** — Same pattern but with
   an RLE block as block 1.

Both lift existing block-level composition theorems to frame level, following the
established proof pattern (derive offset bound, bridge executeSequences via
`ByteArray.empty_append`, call block-level theorem, unfold decompressFrame, handle
dictionary check, substitute and close with grind).

Key difference from the compLit variants: block 2 has numSeq>0, requiring
additional hypotheses for `resolveSequenceFseTables`, `BackwardBitReader.init`,
`decodeSequences`, and `executeSequences`. The `executeSequences` window parameter
uses `block1` directly (bridged from `ByteArray.empty ++ block1`).
