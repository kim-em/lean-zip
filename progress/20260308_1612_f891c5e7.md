# Progress: decompressBlocksWF two-block composition

- **Date**: 2026-03-08 16:12 UTC
- **Session**: f891c5e7 (feature)
- **Issue**: #972

## Accomplished

Added two multi-block composition theorems to `Zip/Spec/Zstd.lean`:

1. **`decompressBlocksWF_two_raw_blocks`**: When a non-last raw block is
   followed by a last raw block, the block loop produces `output ++ block1 ++ block2`.

2. **`decompressBlocksWF_two_rle_blocks`**: Same pattern for RLE blocks.

Both proofs are two-line compositions: `rw [step]; exact single_block`.
No `ByteArray.append_assoc` normalization was needed because `++` is
left-associative in Lean 4, making `output ++ block1 ++ block2` and
`(output ++ block1) ++ block2` definitionally equal.

## Decisions
