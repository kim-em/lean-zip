# Progress: Track C1 — Parametric maxOutputSize for stored block roundtrip

**Date**: 2026-03-06T06:21Z
**Session**: feature (3431618c)
**Issue**: #645

## What was done

Generalized `inflate_deflateStoredPure` from a hard-coded 1 GiB size limit to
a parametric `maxOutputSize` parameter. The theorem now proves:

```lean
theorem inflate_deflateStoredPure (data : ByteArray)
    (maxOutputSize : Nat)
    (h : data.size ≤ maxOutputSize) :
    Inflate.inflate (deflateStoredPure data) maxOutputSize = .ok data
```

This is strictly stronger than the original — it works for ANY `maxOutputSize ≥ data.size`,
not just the default 1 GiB.

## Files changed

- `Zip/Spec/DeflateStoredCorrect.lean`: Generalized theorem signature and proof
  (replaced constant `1024 * 1024 * 1024` with parameter `maxOutputSize`)
- `Zip/Spec/DeflateRoundtrip.lean`: Updated stored case call site to pass
  explicit `maxOutputSize` via wildcard unification

## Key decisions

- The proof change was entirely mechanical: `inflateLoop_deflateStored` was
  already parametric in `maxOutputSize`, so only the top-level theorem needed
  updating
- Used `_` (wildcard) at the `inflate_deflateRaw` call site to let Lean infer
  the default `maxOutputSize` from context, keeping the diff minimal
