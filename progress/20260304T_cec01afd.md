# Malformed Zstd data test expansion

**Issue**: #576
**PR**: #567
**Session**: cec01afd

## What changed

Added 12 new malformed-input test cases to `ZipTest/ZstdNative.lean` (158 lines),
exercising error handling against crafted byte sequences. No implementation changes.

### Frame header tests (5 tests)
- Reserved descriptor bits (3-4) are silently ignored (correct per RFC 8878)
- Content size mismatch: declared 100 bytes, actual 5 → "content size mismatch" error
- Maximum window exponent (31, mantissa 7) parses successfully
- Truncated header: singleSegment=0 but missing window descriptor byte
- Truncated FCS: fcsFlag=3 (needs 8 bytes) with only 1 byte available

### Block and literals tests (7 tests)
- Reserved block type (3) rejected with "reserved block type" error
- Raw block with blockSize=100 but only 3 bytes available → "not enough data" error
- RLE block consumes exactly 1 byte (endPos check with trailing bytes)
- Huffman weights [2,2,1] → implicit last symbol = 3, not power of 2 → error
- Skippable frame with size=4096 but only 5 payload bytes → overflow error
- Valid frame + 2 trailing bytes → "trailing data too short" error
