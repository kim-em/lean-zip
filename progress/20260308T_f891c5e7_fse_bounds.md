## Accomplished

Proved both deliverables from issue #913:

1. **`decodeFseDistribution_size_pos`** — When `decodeFseDistribution` succeeds,
   the output probability array is non-empty (`0 < probs.size`). Composed from
   the existing `decodeFseLoop_invariant` which shows `probs.size ≥ 1` from
   the first iteration.

2. **`decodeFseLoop_probs_ge_neg1`** — When `decodeFseLoop` succeeds and all
   input probs have `.toInt ≥ -1`, all output probs also have `.toInt ≥ -1`.
   This required 8 supporting lemmas covering `Array.push` preservation,
   `pushZeros` bounds, `decodeZeroRepeats` preservation, forward `readBits`
   value bounds, `readProbValue` output bounds, and `Int32.ofNat n - 1 ≥ -1`.

## Key decisions

- Added `hrem_succ : rem + 1 < 2^31` precondition to `decodeFseLoop_probs_ge_neg1`
  because `Int32.ofNat` wraps for values ≥ 2^31, making `ofNat n - 1 ≥ -1` false.
  The FSE spec guarantees `remaining ≤ tableSize ≤ 2^9` so this is always satisfied.

- Used `have hshift : ∀ k, 1 <<< k = 2 ^ k` pattern to convert `<<<` (which goes
  through `HShiftLeft → ShiftLeft → Nat.shiftLeft`) before `simp only` + `omega`.
  `simp only [Nat.shiftLeft_eq]` can't match the typeclass chain directly.

- Used `change` + `intro ⟨i, hi⟩` + `simp only [if_pos/if_neg] at hi ⊢` pattern
  for the `pushZeros` dependent type issue where `rw` fails with "motive not type
  correct" on `if` expressions that appear in `Fin` bounds.
