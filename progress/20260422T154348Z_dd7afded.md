## Decisions

- **Used `grep -Fq -- '<!-- drift-detector:' <<< "$prose"`.** The
  `prose` variable already holds the current inventory line (set by
  `awk -v n="$srcln" 'NR==n{print; exit}' "$INVENTORY"` earlier in
  the same iteration), so the opt-out reuses it instead of
  re-reading the inventory file. Fixed-string, quiet — no regex,
  no stdout noise.
- **Placed the opt-out inside `if (( !found ))`.** The marker only
  matters when a warning would otherwise fire; checking it earlier
  would be wasted work on the 92 lines that already pass cleanly.
- **Left the fourth TODO on line 564 alone.** Per the issue's
  scope-discipline bullet: that row (`Tar.extract` regular-file
  short-read) currently passes pass (c) by coincidence. Adding the
  opt-out marker there would mask a check that is presently
  working.
- **Did not stage the pre-existing worktree-local modifications to
  `.claude/CLAUDE.md` and `.claude/skills/agent-worker-flow/SKILL.md`.**
  Those were present in the worktree before the session began and
  are unrelated to the issue; staging only `scripts/check-inventory-links.sh`
  keeps the PR surgical.
