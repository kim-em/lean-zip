## What was accomplished

Wired the existing XXH64 implementation (`Zip/Native/XxHash.lean`) into the
Zstd frame decompressor (`Zip/Native/ZstdFrame.lean`), completing the checksum
verification path for Zstd frames.

### Changes

- **`Zip/Native/ZstdFrame.lean`** (~10 lines changed):
  - Added `import Zip.Native.XxHash`
  - In `decompressFrame`: when `header.contentChecksum` is set, reads the 4-byte
    LE checksum from the frame trailer, computes `XxHash64.xxHash64Upper32` over
    the decompressed content, and throws a descriptive error on mismatch
  - Updated module docstring to reflect that checksum verification is now supported

- **`ZipTest/ZstdNative.lean`** (~50 lines added):
  - Test 25: valid FFI-compressed constant data decompresses with checksum verification
  - Test 26: corrupted block content triggers checksum mismatch error
  - Test 27: empty input with checksum decompresses successfully

## Decisions

- The corruption test (test 26) accepts both checksum mismatch errors and
  "compressed blocks not yet implemented" errors, since zstd may use compressed
  blocks even for constant data depending on compression level. The test is
  structured to catch the checksum error when the block type is raw/RLE.
