## Decisions

- **File-absolute `eocdPos` at the `findEndOfCentralDir` return
  site**, not `tailStart + pos` in the caller. The returned
  value's semantics now match the archive-layout invariant's
  natural formulation; internal uses of the buffer-relative
  `pos` variable inside `findEndOfCentralDir` are untouched.
- **Keep the `≤ fileSize` check.** The new `≤ eocdPos` check
  subsumes it logically, but the existing `SpanInFile` proof
  obligations name `fileSize` explicitly. Removing the weaker
  guard would force unrelated proof updates. Redundancy is
  harmless in the success path.
- **Extended `build-cd-lh-mismatch.py` rather than forking a
  new script.** The existing script owns the CD/EOCD-mismatch
  family (`eocd-numentries-mismatch.zip`, `eocd-disknum-mismatch.zip`,
  etc.); the new fixture is in the same family. Added
  `eocd_cd_size_override=None` kwarg to preserve byte-identity.
