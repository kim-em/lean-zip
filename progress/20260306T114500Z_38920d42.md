# Progress: Zstd value decoding bounds

- **Date**: 2026-03-06T11:45:00Z
- **Session**: 38920d42 (feature)
- **Issue**: #698

## Accomplished

Proved two characterizing property theorems for Zstd sequence value decoding:

1. **`decodeMatchLenValue_ge_three`**: Any successful match length decoding produces
   a value ≥ 3 (the Zstd minimum match length per RFC 8878 §3.1.1.3.2.1.2).
   - Used `decide_cbv` on a `Fin`-bounded proposition to verify all 53 baselines ≥ 3
   - Needed a Nat-indexed wrapper to bridge the Fin-based decide_cbv proof with
     the Array.getInternal used in the main theorem's goal (omega can't unify
     GetElem-based and getInternal-based array accesses)
   - Final proof uses `Nat.le_trans` for exact unification instead of omega

2. **`decodeOffsetValue_ge_two`**: Non-repeat offset codes (code ≥ 1) produce
   values ≥ 2. Follows from `2^code ≥ 2` when `code ≥ 1`, proved via
   `Nat.pow_succ` + `Nat.one_le_two_pow`.

## Key patterns discovered

- **omega vs exact for array access**: omega treats `Array.getElem` (via GetElem
  typeclass) and `Array.getInternal` (after unfold) as different expressions.
  Use `exact` with `Nat.le_trans` for definitional unification instead.
- **decide_cbv on Fin-bounded propositions**: Works well for verifying properties
  of all entries in a concrete array. Must use `Fin` (not `Nat` with bound) to
  get a closed proposition that decide_cbv can handle.
