## Accomplished

1. **`decompressFrame_two_compressed_literals_blocks_content`**: Frame-level theorem
   proving that when `decompressFrame` succeeds on a frame with two compressed blocks
   (both numSeq=0), the output equals `literals1 ++ literals2`. ~80 lines of statement
   + 20 lines of proof.

2. **`decompressFrame_two_compressed_sequences_blocks_content`**: Frame-level theorem
   proving that when `decompressFrame` succeeds on a frame with two compressed blocks
   (both numSeq > 0), the output equals `blockOutput1 ++ blockOutput2`. ~120 lines of
   statement + 30 lines of proof.

## Key decisions

- **Inlined step composition**: Instead of calling the pre-composed
  `decompressBlocksWF_two_compressed_*` theorems directly (which fail due to
  dependent-type mismatch in Huffman `if let` threading), used the step + single-block
  theorem composition with `cases huffTree1` to eliminate the match. This pattern
  matches the existing `decompressFrame_compressed_seq_then_compressed_lit_content`.

- **Simplified Huffman threading in sequences theorem**: Used `huffTree1` directly
  in `hlit2` (not `if let some ht := huffTree1 then some ht else none`) since
  they're definitionally equal. Converted with `(by cases huffTree1 <;> exact hlit2)`.

- **Window check bridging**: Both theorems needed `hexec'` conversions. The literals
  theorem needed only `hexec1'` (ByteArray.empty window simplification). The sequences
  theorem needed both `hexec1'` and `hexec2'` (ByteArray.empty ++ blockOutput1 →
  blockOutput1 via `ByteArray.empty_append`).
