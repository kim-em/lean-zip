# Track E: FSE backward bitstream reader and symbol decoding

**Session**: 2a3619ba
**Issue**: #431
**Type**: feature

## What was done

1. Added `BackwardBitReader` structure to `Zip/Native/Fse.lean` (~65 lines):
   - Reads bits MSB-first from Zstd's backward bitstream format (RFC 8878 §4.1)
   - `init`: finds sentinel bit in last byte, sets up initial state
   - `readBits`: reads n bits MSB-first, moving from end toward beginning
   - `isFinished`: checks if all meaningful bits consumed
   - `highBitPos`: pure if-else chain (no mutable state outside do blocks)

2. Added `decodeFseSymbols` function (~25 lines):
   - Initializes FSE state from the backward bitstream
   - Decodes symbols via table lookup: `cell = table[state]`, emit symbol,
     read `numBits` bits, compute `newState = baseline + readBits`
   - Skips the final state transition after the last symbol

3. Added 6 new tests in `ZipTest/FseNative.lean` (~130 lines):
   - Test 9: BackwardBitReader init and multi-width bit reads
   - Test 10: Sentinel-only byte (moves to previous byte)
   - Test 11: FSE decode round-trip with 2-symbol table
   - Test 12: Error on empty range
   - Test 13: Error on zero last byte (no sentinel)
   - Test 14: Integration with real Zstd-compressed data

4. Updated module docstring to reflect the three components

## Decisions

- Used `Inhabited` instead of `Repr` for `BackwardBitReader` since `ByteArray`
  doesn't derive `Repr`
- Replaced `let mut` loop in `highBitPos` with explicit if-else chain to avoid
  mutable state outside a `do` block
- `decodeFseSymbols` skips the final bit read after the last symbol (standard
  FSE behavior — the last symbol's state bits are not consumed)
