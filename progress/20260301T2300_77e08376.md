## What was done

Repaired all 3 proofs that broke when `decodeHuffman.go` (native) and
`decodeSymbols` (spec) were converted from fuel-based to well-founded
recursion in PRs #341 and #337.

### 1. `decodeHuffman_go_append` (InflateRawSuffix.lean)

Suffix invariance proof. Key fix: omega couldn't relate
`(brAppend br₁ suffix).bitPos` and `br₁.bitPos` despite being
definitionally equal. Added explicit `have hbp₁ := rfl` witnesses.
Used `split` for decidable if conditions instead of `dif_neg`.

### 2. `decodeHuffman_correct` (DecodeCorrect.lean)

Native → spec correspondence. Added helper `specTable_cw_nonempty`
(proving codewords in allCodes tables are nonempty, needed for
`decode_shorter`). Proof uses `termination_by dataSize * 8 - br.bitPos`
matching the native function's WF recursion. Each recursive case proves
the WF guard for `decodeSymbols` via `decode_shorter` and
`readBitsLSB_some_length`.

### 3. `decodeHuffman_complete` (DecodeComplete.lean)

Spec → native correspondence. Most complex of the three — needed to show
the native WF guards pass (bitPos advances, stays within dataSize bounds).
Added `hple` and `hdata` hypotheses to the theorem statement (the original
statement was false for small `dataSize`; these hold trivially when
`dataSize = br.data.size` as in the actual API). Made
`specTable_cw_nonempty` non-private; added `BitReaderInvariant` import.
Uses `termination_by br.toBits.length`.
