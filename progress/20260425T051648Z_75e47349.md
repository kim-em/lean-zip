## Notes

- The issue suggested a `unameOverride : Option String` helper-data
  struct field. The actual builder script has no helper-data struct —
  each function calls `Tar.buildHeader` with an inline `Entry` literal.
  Since `Binary.writeString` writes UTF-8 bytes verbatim (including
  U+0000), setting `entry.uname := "trusted\x00rogue"` directly
  achieves the same effect as the proposed override and avoids
  introducing an unused parameter on `Tar.buildHeader`. This matches
  the `linkname` precedent (`buildUstarLinknameNulInName` sets
  `entry.linkname` directly without an override).
