Added two two-block composition theorems to `Zip/Spec/Zstd.lean`:

1. **`decompressBlocksWF_compressed_literals_then_raw`** — non-last compressed
   block (numSeq=0) + last raw block → `output ++ literals1 ++ block2`
2. **`decompressBlocksWF_compressed_literals_then_rle`** — non-last compressed
   block (numSeq=0) + last RLE block → `output ++ literals1 ++ block2`

Both follow the established pattern: `rw` with the step theorem, then `exact`
with the single-block theorem. Raw/RLE blocks don't consume Huffman state, so
the `prevHuff` argument to block 2's theorem is a wildcard `_`.

## Decisions

  pattern), since the single-block theorems for raw/RLE don't need the
  `if let some ht := huffTree then some ht else prevHuff` state threading.
