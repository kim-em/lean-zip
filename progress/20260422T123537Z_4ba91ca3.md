## Accomplished

Six new characterizing-property theorems for the project's checksum
families, extending the `_append` template established by PR #1640
(`crc32_append`) and PR #1641 (`adler32_append`):

1. `Crc32.Spec.checksum_empty : checksum [] = 0` ‚Äî `@[simp]`, proved by
   `simp only [checksum, updateList_nil]; bv_decide`.
2. `Adler32.Spec.checksum_empty : checksum [] = 1` ‚Äî `@[simp]`, proved
   by `simp only [checksum, updateList_nil, pack, init]; decide`.
3. `Adler32.Spec.checksum_singleton (b) : checksum [b] = UInt32.ofNat ((1 + b.toNat) * 65537)`.
   Closed form follows from `1 + b.toNat < 256 < 65521` (no modular
   reduction fires) and `pack (a, a) = a + a*65536 = a*65537` for
   `a < 65536`. Bridged through `‚Üê UInt32.toNat_inj` and
   `pack_toNat_of_bounds`. Final arithmetic step closed by `omega`.
4. `Crc32.Native.crc32_empty (init) : crc32 init ByteArray.empty = init`
   ‚Äî `@[simp]`, both branches of the `init == 0` guard collapse via
   `bv_decide` after the foldl over an empty array reduces.
5. `Adler32.Native.adler32_empty (init) : adler32 init ByteArray.empty = init`
   ‚Äî `@[simp]`, reduces to `Spec.pack_unpack init` once `updateBytes`
   over an empty `ByteArray` is dispatched.
6. `Adler32.Native.adler32_singleton (b) : adler32 1 (ByteArray.push ByteArray.empty b) = ...`
   ‚Äî bridges to `Spec.checksum_singleton` after rewriting
   `(ByteArray.push ByteArray.empty b).data.toList = [b]` (rfl) and
   `Spec.unpack 1 = Spec.init` (`decide`).

The CRC32 single-byte case is **deliberately omitted** ‚Äî it expands to
8 `crcBit` calls with no clean closed form; stating it as
`checksum [b] = crcByte 0xFFFFFFFF b ^^^ 0xFFFFFFFF` would be
tautological (restating the definition). See issue #1654 *"Scope
discipline"* block.

## Decisions / patterns

- **`UInt32.toNat_ofNat` vs `UInt32.toNat_ofNat_of_lt'`.** The plan's
  draft proof used `UInt32.toNat_ofNat`, which simp-rewrites
  `(OfNat.ofNat n).toNat = n % 2^32` (note: `OfNat`, not `UInt32.ofNat`).
  Since `checksum_singleton`'s RHS is literally
  `UInt32.ofNat ((1 + b.toNat) * 65537)`, the right lemma is
  `UInt32.toNat_ofNat_of_lt'`, which rewrites `(UInt32.ofNat n).toNat = n`
  given `n < UInt32.size`. This sidesteps the mod-then-`Nat.mod_eq_of_lt`
  detour from the plan's draft.
- **No `pack_toNat_of_bounds` visibility bump needed.** The plan
  flagged that this `private` helper might need promotion to `protected`
  for `checksum_singleton`. Both are in `Adler32.Spec` and the same file,
  so `private` is sufficient ‚Äî no signature change.
- **Final arithmetic via `omega`.** The closing step
  `1 + b.toNat + (1 + b.toNat) * 65536 = (1 + b.toNat) * 65537` is
  linear in `b.toNat` with constant coefficients, so `omega` handles it
  without needing `ring` (which the project avoids).
- **`Spec.checksum_singleton` lifts to native via defeq.** After
  rewriting the data list to `[b]` and `Spec.unpack 1` to `Spec.init`,
  the goal `pack (updateList init [b]) = ...` is definitionally equal
  to `checksum [b] = ...`, so a bare `exact Spec.checksum_singleton b`
  closes it ‚Äî no need for an intermediate `updateList_unpack_init`
  helper as the plan tentatively suggested.
