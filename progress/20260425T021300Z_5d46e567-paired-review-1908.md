### A.3 Smuggled value choice: `totalEntries = 99`

`totalEntries` is a `UInt16` field at standard-EOCD offset `+10`
(per APPNOTE §4.3.16 "Total number of entries in the central
directory"). The APPNOTE §4.3.16 ZIP64-indirection sentinel is
`0xFFFF = 65535`.

The fixture's choice of `99` is well-defended:

- `99 ≠ 0xFFFF`: so the "relaxed sentinel" arm at :402 fails on its
  first disjunct (`stdTotalEntries16 == val16Max`).
- `99 ≠ 1` (the actual entry count, carried in the ZIP64 EOCD64
  `totalEntries` at offset `+32`): so the second disjunct
  (`stdTotalEntries16.toNat == totalEntries`) also fails.
- `99` is also not numerically close to the ZIP64 override, avoiding
  any accidental off-by-one or sign-conversion coincidence.
- `99` is small enough to fit `UInt16` with no byte-order ambiguity
  (`0x0063`), matching the `cdSize=99` / `diskWhereCDStarts=99` /
  `numberOfThisDisk=99` sibling convention across the wave.

The value `99` is deliberately consistent across the per-slot wave
(`cdSize=99` in PR #1900, `diskWhereCDStarts=99` in PR #1911,
`numberOfThisDisk=99` in in-flight PR #1902, `totalEntries=99` here) —
a uniform smuggling convention that makes the family's fixture intent
and inter-fixture comparisons easy to read.

## F. Reviewer observations

- **F.1 Smuggling-vector emphasis.** The feature-side PR body
  (commit message + test comment at :855-859 + inventory narrative
  at :1268) each note that `totalEntries` is a "particularly notable
  smuggling vector" because it controls the CD-walker's iteration
  loop. This emphasis is appropriate: a relaxed reader that trusts a
  smuggled `totalEntries=99` value would attempt to walk 99 CD
  entries rather than the actual 1, potentially spilling into the
  EOCD region (if the EOCD itself isn't validated) or silently
  truncating (if the EOCD is validated but the CD-walk loop
  under-shoots). The CD-walker at
  [`Zip/Archive.lean:626-679`](/home/kim/lean-zip/Zip/Archive.lean:626)
  (approximately — confirm by grep) uses `totalEntries` as its upper
  loop bound, so the smuggling vector is real and well-flagged.

- **F.2 UInt16 overflow-convention consistency.** `totalEntries` is
  UInt16 at standard-EOCD position `+10`; the ZIP64 override at
  EOCD64 position `+32` is UInt64. Per the per-slot-family convention,
  all three of the UInt16 standard-EOCD fields (`totalEntries`,
  `numberOfThisDisk`, `numEntriesThisDisk`, `diskWhereCDStarts`) use
  `val16Max = 0xFFFF` as the sentinel, while the two UInt32 fields
