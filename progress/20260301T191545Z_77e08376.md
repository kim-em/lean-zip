## Accomplished

Proved three structural properties in `Zip/Spec/DeflateSuffix.lean` that were
sorry-patched after PR #344 converted `decode.go`/`decode.goR` from fuel-based
to well-founded recursion:

1. **`decode_go_suffix`**: Suffix invariance — appending byte-aligned bits to
   the input preserves the decoded result. Uses `decodeStored_append`,
   `decodeSymbols_append`, and `decodeDynamicTables_append` for the sub-cases.

2. **`decode_goR_fst`**: Consistency — `decode.goR` agrees with `decode.go`
   on the decoded result (i.e. the first component of the pair).

3. **`decode_goR_exists`**: Existence — if `decode.go` succeeds, then
   `decode.goR` also succeeds with some remaining bits.

## Decisions

- **Induction principle**: All three proofs use `Nat.strongRecOn` on

  in `Deflate.lean`. The functional induction principle `decode.go.induct` was
  not available.

- **WF recursion goal shape**: After `simp [hf]` in the non-final-block case,
  the goal takes the form `bits'.length < bits.length ∧ decode.go (bits' ++ suffix) acc' = some result`
  (the conjunction includes the WF guard). This is handled by `exact ⟨hblen, ih ...⟩`.

- **Helper lemmas**: All required suffix-invariance lemmas for sub-functions
  (`decodeStored_append`, `decodeSymbols_append`, `decodeDynamicTables_append`,
  `readBitsLSB_append`) already existed in the file.
