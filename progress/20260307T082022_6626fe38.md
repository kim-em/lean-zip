## Accomplished

- Proved `buildFseTable_numBits_le`: every cell in the FSE decoding table
  has `numBits.toNat ≤ accuracyLog`
- This is condition (c) of the `ValidFseTable` predicate
- Introduced `set!_preserves_forall` helper for reasoning about
  `Array.set!` preserving Fin-indexed predicates through dependent type issues

## Approach

Followed the same `forIn_range_preserves` pattern as the existing
`buildFseTable_cells_size` proof:
- Decompose `buildFseTable` into 4 loop results via `Except.bind_eq_ok'`
- Track per-cell property through loops 1 (place -1 symbols),
  2 (distribute symbols), skip loop 3 (only modifies symbolCounts),
  and loop 4 (compute numBits/newState)
- Key arithmetic: `(al - log2 nextState).toUInt8.toNat ≤ al` by
  `Nat.mod_le` composed with `Nat.sub_le`

## Decisions

- Used Fin-indexed predicates instead of getElem! to avoid Inhabited
  typeclass issues with generic helpers
- The `set!_preserves_forall` helper uses `▸` (subst) to handle
  dependent type rewriting through `getElem_setIfInBounds_self/ne`
- Did not need a separate `buildFseTable_numBits_zero_initial` helper
  (the loop invariant approach handles the initial state inline)
