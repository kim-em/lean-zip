**F-c (#1653).** `ZipTest/FuzzInflate.lean` gained `tryStreamingGzip` /
`tryStreamingRawDeflate` probes that drive the Lean-side
`IO.Ref UInt64` counter + `next > maxDecompressedSize` guard added by
#1610 and default-flipped by #1631. A `maxClasses : Array UInt64`
picks one of `#[0, 64, 1024, 65536]` per iteration ‚Äî `0` preserves
coverage of the unlimited branch after the 1 GiB default flip. The
non-obvious subtlety is the `chunksReadStream` implementation: it
skips exhausted/empty chunks because the `decompressStream` top-level
loop treats empty reads as EOF, and the fuzz driver wants one chunk
per read without premature termination.

**F-d (#1657).** `Zip/Gzip.lean:93-97` and `Zip/RawDeflate.lean:64-68`
now read `if next < total || (maxDecompressedSize ‚âÝ 0 && next >
maxDecompressedSize)`. The `next < total` disjunct detects `UInt64`
wrap-around directly (without the `0xFFFFFFFFFFFFFFFF` magic constant
the plan's draft alternative would have needed). Auditors #1667 ¬ß1
verified correctness case-by-case; ¬ß6 confirmed both disjuncts feed
the same `"exceeds limit"` throw so the `error-wording-catalogue`
discipline is preserved. No new tests: the overflow branch is
unreachable at realistic input sizes
(‚âà 584 years at 1 GB/s sustained), and adding reach would require
exposing the `IO.Ref UInt64` state (a structural gut for a zero-caller
branch ‚Äî #1667 ¬ß4 accepts the no-test argument explicitly).

**P5.2 review (#1656).** Quality audit of the merged P5.2 block

would now fail loudly instead of being swallowed; sanitizer traps
and panics are process-level and bypass IO catches regardless.

### 3. Characterizing-property pivot ‚Äî six new theorems

The `_append` characterizing-property pattern from PRs #1640 (CRC32)
and #1641 (Adler-32) continued in two shapes:

- **#1650** (review of #1640/#1641) reduced the proof bodies
  for `crc32_append` / `adler32_append` by 18 net lines: dropped the
  `xor_twice` helper in favour of `UInt32.xor_assoc` + `.xor_self` +
  `.xor_zero` library simp lemmas, collapsed repeated
  `updateBytes_eq_updateList` rewrites into single `simp only` calls,
  inlined single-use `hvalid` witnesses via `h ‚ñ∏ term`. Also removed
  redundant `example` stanzas in `ZipTest/NativeChecksum.lean` (per
  CLAUDE.md's *"theorems are stronger than finite test sets"*).
- **#1661** (feature) added six new characterizing theorems ‚Äî
  `Spec.checksum_empty` for both checksums (CRC32 returns `0`,
  Adler-32 returns `1`), `Adler32.Spec.checksum_singleton (b)`
  giving the closed form `(1 + b.toNat) * 65537` (proved via
  `pack_toNat_of_bounds` from #1641, demonstrating the helper's
  re-usability beyond its first consumer), and the three `Native`
  wrappers (`crc32_empty`, `adler32_empty`, `adler32_singleton`)
  that lift the spec theorems through `updateBytes_eq_updateList` /
  the native `init == 0` guard collapse.

The **CRC32 single-byte case was deliberately omitted** as
tautological ‚Äî it would expand to 8 `crcBit` calls with no clean
closed form, and stating `checksum [b] = crcByte 0xFFFFFFFF b ^^^
0xFFFFFFFF` restates the definition rather than characterising it.
Issue #1664 (filed by the planner concurrently with #1668; still
unclaimed at this summarize's write time) extends the pattern to
`checksum_pair` ‚Äî the two-byte closed form that `checksum_empty` and
`checksum_singleton` naturally compose with.
