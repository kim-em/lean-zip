# Progress: Track E — Zstd XxHash64 specification predicates

- **Date**: 2026-03-05 UTC
- **Session type**: feature
- **Issue**: #595

## Accomplished

Created `Zip/Spec/XxHash.lean` with formal specification predicates for the
XXH64 hash function used by Zstd content checksums (RFC 8878 §3.1.1).

### Predicates (6)
- `ValidPrimes`: the five 64-bit prime constants match the xxHash spec
- `RoundState`: four-accumulator state during stripe processing
- `initState`: initial accumulator state for a given seed
- `ValidRound`: single round step predicate
- `ValidConvergence`: four-accumulator convergence predicate
- `ValidAvalanche`: final 3-step mixing predicate
- `ValidProcessRemaining`: remaining-bytes processing predicate

### Theorems (4, 1 sorry)
- `primes_valid`: proved (trivial by rfl)
- `xxHash64_deterministic`: proved (rfl — pure function)
- `xxHash64Upper32_eq`: proved (unfold + rfl)
- `xxHash64_empty`: **sorry** — requires unfolding `Id.run do` with while loops
  in `processRemaining`; the while loops use `ForIn`/`Range` which don't reduce
  via `simp`/`unfold` when the loop body involves mutable state

### Quality metrics
- Starting sorry count: 5 (3 in Fse.lean, 2 in Zstd.lean)
- Ending sorry count: 6 (+1 in XxHash.lean for `xxHash64_empty`)
- `lake build`: passes

## Decisions
- Spec level: tier 3 (algorithmic correspondence) — xxHash is defined by its
  algorithm, not by mathematical properties. No useful algebraic properties exist.
- Left `xxHash64_empty` as sorry rather than fighting with `Id.run do` while loop
  unfolding. A dedicated lemma about `processRemaining` with zero-length input
  would make this provable, but requires Lean 4 loop reasoning infrastructure.
