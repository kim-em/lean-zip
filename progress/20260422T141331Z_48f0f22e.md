## Proof notes

The Spec proof mirrors `checksum_singleton`'s shape exactly:

1. `rw [← UInt32.toNat_inj]` drops into `Nat` equality.
2. `simp only [checksum, updateList, List.foldl_cons, List.foldl_nil,
   updateByte, init, MOD_ADLER]` unfolds one layer of `updateByte` per
   byte, exposing `(... % 65521)` patterns on both state components.
3. Three `Nat.mod_eq_of_lt` lemmas (`ha₁`, `ha₂`, `hb₂`) eliminate the
   mods — the four state values all fit under `65521` (max `s.2` after
   two updates is `767`).
4. `rw [ha₁, Nat.zero_add, ha₁, ha₂, hb₂, pack_toNat_of_bounds ...,
   UInt32.toNat_ofNat_of_lt' ...]` collapses the chain.
5. `omega` closes the final `Nat`-arithmetic congruence
   `(1 + b₁) + (1 + b₁ + b₂) = 2 + 2·b₁ + b₂`.

The `rw [ha₁, Nat.zero_add, ha₁, ...]` cadence (rather than four
separate `have` hypotheses, one of which was `(0 + x) % m = x`) is the
same pattern `checksum_singleton` uses — `Nat.mod_eq_of_lt` only proves
`n % m = n`, so the `0 + ...` form must be normalised to `...` via
`Nat.zero_add` before the second `ha₁` rewrite can fire.

The Native proof is a four-line bridge: expand `ByteArray.push`'s
`data.toList` to `[b₁, b₂]` by `rfl`, unpack the initial state via
`Spec.unpack 1 = Spec.init` (`by decide`), drop to `Spec` via
`updateBytes_eq_updateList`, and apply `Spec.checksum_pair`.

## Decisions made

- **Did not promote `pack_toNat_of_bounds` from `private` to
  `protected`.** #1661 already merged `checksum_singleton` alongside
  the `private` version, so the contingency in the issue body is a
  no-op — since `checksum_pair` lives in the same file/namespace,
  `private` access works.
- **Did not add `@[simp]`** to either theorem. Matches the
  `checksum_singleton` convention: closed forms aren't simp-normal.
- **Did not extract a `mod_id_of_lt_adler` helper.** The Spec proof
  came in at 24 LOC (including the docstring), well under the 30 LOC
  threshold the issue set for extraction.
- **Did not add a `#eval`-based sanity check.** The reference-vector
  check mentioned in the issue was development-time guidance only
  ("must not ship").
