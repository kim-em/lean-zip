### Predicates (all with `Decidable` instances)
- `ValidSequence`: single sequence has valid literal length, positive offset, and
  offset within reach of output
- `ValidSequenceList`: total literal consumption ≤ available literals
- `ValidOffsetHistory`: exactly 3 entries, all positive (uses `get!` to match
  implementation access pattern)

### Proven theorems
- `resolveOffset_history_size`: history size preserved across all branches (by `rfl`
  for `#[_, _, _]` constructors, `exact h` for unchanged history)
- `resolveOffset_positive_large`: rawOffset > 3 → resolved offset > 0
- `resolveOffset_positive_litLen_pos`: with valid history and litLen > 0, resolved
  offset > 0 (covers repeat offset codes 1–3 and actual offsets ≥ 4)
- `initial_history_valid`: `#[1, 4, 8]` satisfies `ValidOffsetHistory` (by `decide`)

## Decisions
- Used `get!` (matching implementation's `history[i]!`) instead of `Fin`-indexed `get`
  for `ValidOffsetHistory`. This avoids a `get!` vs `get` mismatch in proofs.
- `resolveOffset_positive_litLen_pos` requires `litLen > 0` because the `rawOffset = 3,
  litLen = 0` case gives `history[0] - 1` which can be 0.
- Used `rcases rawOffset` for case analysis in the positivity proof to avoid issues
  with match fallback branches not reducing `Prod.fst`.
