## What landed

`computeCodeLengths_complete` (gold-standard characterizing property): for
`freqs` with ≥ 2 distinct nonzero-frequency symbols, all `< numSymbols`, with
`numSymbols ≤ 2^maxBits`,

    kraftSum ((computeCodeLengths freqs numSymbols maxBits).filter (· != 0)) maxBits
      = 2 ^ maxBits

i.e. the encoder hot path provably emits a *complete* code (Kraft-exact, not
just `≤`), the equality zlib's inflate requires. Companion to
`computeCodeLengths_valid` (the `≤` half). Plus deliverable 2,
`fixKraftList_limitedPairs_id`: `fixKraftList` is the identity on the
multi-symbol output (overflow branch proven dead). The 1-symbol RFC 1951
boundary is documented in the `≥ 2` hypothesis.

## Proof structure (new lemmas, all in `Zip/Spec/HuffmanEncode.lean`)

1. **`repairBl_count_loop` / `repairBl_count`** — `repairBl` preserves
   `blCountSum` under the same invariants that drive `repairBl_loop` (each
   iteration is a well-defined `repairStep`, which conserves leaf count).
2. **`huffman_leaf_count`** (+ `depths_length_eq`, `insertByWeight_depths_sum`,
   `buildHuffmanTree_depths_length`, `sum_map_one`) — the Huffman tree built
   from a frequency list emits exactly one `(symbol, depth)` pair per input
   entry; merging preserves the total leaf count.
3. **`filter_set_perm` / `foldl_set_filter_perm`** — folding `set` over
   distinct, in-range symbol positions (from an all-zero accumulator) and
   dropping zeros yields a *permutation* of the assigned lengths.
4. **`assignLengths_zip_expandBl_perm`** (+ `zip_append_map_snd`,
   `replicate_zero_getElem`) — the `assignLengths` of the `limitedPairs` zip,
   filtered, is a permutation of `expandBl bl`; the `maxBits` padding is
   provably unused because `|syms| = blCountSum bl = |expandBl bl|`.
5. **`assignLengths_zip_tree_kraft`** — internalizes
   `cappedBlCount_repairBl_complete` (histogram completeness) and
   `repairBl_count` so the symbol side only needs distinctness/range; bridges
   `kraftSum (filter …) = kraftSum (expandBl bl) = blKraft bl = 2^maxBits`.
6. **`nodup_lt_length_le`** — a `Nodup` list of naturals `< n` has length `≤ n`
   (pigeonhole via `erase`), used to derive `nz.length ≤ numSymbols ≤ 2^maxBits`.

## Decisions / patterns

- No Mathlib: avoided `set` (used `generalize … = nz` + `rw … at` to abstract
  the nonzero list past branch navigation), and `List.Subperm` (absent in
  v4.30 core — proved the nodup-length bound by erase-induction instead).
- The `let`-naming problem in `limitedPairs` was solved by *not* spelling the
  giant `bl`/`syms` terms: `simp only [limitedPairs]` exposes the concrete zip
  and unification fills `syms`/`t` in `assignLengths_zip_tree_kraft`.
- Added a non-vacuity `example` instantiating the theorem on a concrete input.
