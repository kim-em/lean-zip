**Issue**: #1311

## Summary

Audited `Zip/Spec/LZ77.lean` (554 lines) and `Zip/Spec/LZ77Lazy.lean` (314 lines)
for proof quality. The issue expected ~27 bare `simp` calls in LZ77.lean and ~11 in
LZ77Lazy.lean, but these had already been cleaned up by a prior PR. Focused on
tactic merging, proof compression, and lemma reuse instead.

## Changes

### LZ77.lean (net -12 lines)

- **Merged consecutive `rw` calls**: Combined separate `rw` invocations on the same
  target into single calls (3 sites in `take_append_copied_eq_take`)
- **`resolveLZ77_findMatch_step` refactored**: Replaced manual `resolveLZ77` unfolding
  + boolean guard manipulation (8 tactic steps) with `resolveLZ77_reference_valid`
  reuse (4 steps). This is the key improvement — the proof now delegates the
  validity check to an existing lemma instead of re-deriving it inline
- **`dsimp only` over `simp only`**: Replaced `simp only` with `dsimp only` where
  only definitional reduction was needed (Fin.val in `take_append_copied_eq_take`,
  `findLongestMatch_end_le`)
- **`Option.some.inj`**: Replaced `simp only [Option.some.injEq] at hmatch; exact hmatch.symm`
  with `exact (Option.some.inj hmatch).symm`
- **Compressed 3-line case to term proof**: `matchLength.go_in_bounds` case using `▸`

### LZ77Lazy.lean (net -1 line)

- Removed unused `rename_i hlen3` in `matchLZ77.go_length_le`
- Remaining proofs are structurally necessary with minimal compression opportunity
