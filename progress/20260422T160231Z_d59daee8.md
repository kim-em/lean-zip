## Proof shape

Strengthened invariant over a free starting `(a, bsum)`:

    ∀ m a bsum,
      a + m·b < 65521 →
      bsum + m·a + b·T_m < 65521 →
      updateList (a, bsum) (replicate m b)
        = (a + m·b, bsum + m·a + b·T_m)

Induction on `m`; in the `succ k` branch:
1. A local `change … at hb` and `show …` normalize `k + 1 + 1`
   to `k + 2` so `omega` sees matching atoms.
2. Four distributivity `have`s (`hka`, `hkb`, `hkab`, `hbtri`)
   expose non-linear products as linear hypotheses for `omega`.
3. `hbyte` computes `updateByte (a, bsum) b` under the bounds.
4. The IH fires with `(a + b, bsum + (a + b))`; bounds discharge
   by `omega` using the distributivity `have`s.
5. The final component equalities close with
   `Prod.mk.injEq .. |>.mpr ⟨?_, ?_⟩ <;> omega`.

Instantiate at `(1, 0)` and compose with `pack_toNat_of_bounds`
exactly as `checksum_replicate_zero` does.

The triangular recurrence `(m+1)·(m+2)/2 = m·(m+1)/2 + (m+1)`
is proved locally via `Nat.add_mul_div_right` — no Mathlib
dependency needed, as the planner anticipated.

## Decisions and observations

- **Strengthened-invariant choice**. Used the full
  `(a, bsum)`-free form from the issue's sketch. A simpler
  `a = 1, bsum` free form would not have sufficed because the
  A-component changes as soon as `b ≠ 0`.
- **`Nat.mul_succ` needs explicit arguments**. A terser
  `rw [htri k, Nat.mul_add, Nat.mul_succ, Nat.mul_comm b.toNat k]`
  fails — the bare `Nat.mul_succ` matches `k * (k + 1)` before
  `b.toNat * (k + 1)` and expands the wrong subterm. Explicit
  `Nat.mul_succ b.toNat k` pins the rewrite to the intended
  occurrence. Worth noting as a skill-level gotcha for future
  proofs that reach inside a `b.toNat * (x * (x + 1) / 2)` shape.
- **`change` vs `show`**. Needed both: `change … at hb` to
  normalize the hypothesis (because `omega` cannot unify
  `k + 1 + 1` with `k + 2` through the `*` atoms), and `show`
  to force the goal into the same form.
- **No `@[simp]` tag**. Two hypothesis gates, matching the
  existing convention (the planner's table).
