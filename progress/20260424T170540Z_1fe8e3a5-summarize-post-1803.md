## Attack-class closures

The 6 feature PRs each close an independent CD-parse-time defensive
dimension on the CD entry or EOCD64 record:

1. **CD `versionNeededToExtract` upper bound (#1807).** Rejects CD
   entries claiming `versionNeededToExtract > 45` (the APPNOTE 6.3
   ZIP64 ceiling) at `parseCentralDir`
   ([Zip/Archive.lean:546](/home/kim/lean-zip/Zip/Archive.lean:546)),
   before the existing CD-vs-LH one-sided downgrade check (PR #1736)
   can be bypassed by both sides claiming an unsupported value.
2. **Per-entry `localOffset + 30 ≤ cdOffset` archive-layout
   invariant (#1813).** Per-entry micro-counterpart of the in-flight
   archive-level macro `cdOffset + cdSize ≤ eocdPos` guard (issue
   #1799 / PR #1809). Closes the parser-differential window where
   `Archive.list` accepts an entry whose LH position falls inside or
   past the CD region.
3. **`internalFileAttributes` reserved-bits guard (#1819).** Rejects
   CD entries with any bit other than bit 0 set in APPNOTE §4.4.10
   `internalFileAttributes` (CD +36). Mask `internalAttrs &&& 0xFFFE
   == 0` preserves Info-ZIP text-flag interop while rejecting
   smuggled reserved bits.
4. **CD flag bit-5 (compressed patched data) rejection (#1824).**
   Closes the third independent GPF bit dimension (siblings #1818
   bit 4 enhanced deflating, #1822 bits 7-10/12/14/15 reserved/unused
   are in flight at batch close). Once those land, the GPF word is
   fully validated except bits 1-3 (deflate/data-descriptor) and bit
   11 (UTF-8), all of which are legitimate.
5. **EOCD64 `versionMadeBy` spec-version upper bound (#1826).**
   Archive-level counterpart to the per-entry CD `versionMadeBy >
   63` guard (issue #1812 / in-flight PR #1820); the pair closes the
   `versionMadeBy` upper-bound dimension across both ZIP layers.
6. **CD entry name NUL-byte rejection (#1831).** Rejects raw CD
   `nameBytes` containing `0x00` before the UTF-8 / Latin-1 decode.
   Closes the classic NUL-truncation parser-differential smuggling
   vector simultaneously on `Archive.list` (silent NUL-path
   surfacing) and `Archive.extract` (POSIX `open` truncates at NUL).
