## #2571 — skipped (blocked on #2536)

`feat: reject incomplete dynamic-block Huffman codes in native inflate`.

The native change (deliverable 1) is easy, but it cannot land green. Rejecting
incomplete codes in `decodeDynamicTrees` falsifies the proven
**complete-direction** chain:
`decodeDynamicTrees_complete` → `inflateLoop_complete` → `inflate_complete`
→ `inflate_deflateDynamicBlock`. These prove *spec-accepts ⟹ native-accepts*,
but the spec `Deflate.Spec.decodeDynamicTables` guards only on
`Huffman.Spec.ValidLengths` = the Kraft **inequality** (`≤`), so it *accepts*
incomplete codes. Making native reject them breaks the chain.

The issue says to "thread a completeness fact", but none exists in the current
tree: `deflateDynamicBlock_spec` emits only `ValidLengths`, not completeness
(`kraftSum = 2^maxBits`). A green landing needs either tightening the spec or
threading completeness up the chain — both bottom out at proving the encoder
emits complete codes (`computeCodeLengths_complete`, **#2536**, OPEN/replan),
including the RFC single-/empty-distance-code exception (a literals-only block
emits an empty or single distance code; `computeCodeLengths_complete` requires
≥2 distinct nonzero-freq symbols and explicitly excludes the single-code case).
Adding an unproven hypothesis to the public dynamic roundtrip would weaken a
proven theorem (forbidden by CLAUDE.md).

Recommended replan: gate with `depends-on: #2536`. Detailed analysis posted as
a comment on #2571; skipped → `replan`.

## #2573 — completed

`test: adversarial inflate-decode fuzz target`.

Added `runDecodeFuzz` to `ZipTest/FuzzInflate.lean` (+183 lines, pure test
code). A second decode-side census complementing `runDiffFuzz`'s single-bit
header mutations, over three deterministic (fixed-seed) input classes, 256
iters each:

1. **Truncated** raw-DEFLATE streams cut at a random byte offset (mid-block,
   since DEFLATE is bit-packed) → native `Inflate.inflate` vs FFI
   `RawDeflate.decompress`.
2. **Concatenated gzip members** (1–4 back-to-back, ½ chance of a truncated
   tail) → native `GzipDecode.decompress` vs FFI `Gzip.decompress`. Untruncated
   streams asserted to round-trip through both; truncated folded into census.
3. **Random byte streams** smoke layer → native raw inflate vs FFI.

`DecodeCensus` tallies the four verdict buckets. Silent corruption (both accept,
bytes differ) fails; native-accept/FFI-reject recorded, not failed.

Census (seed `0xc0deba5e`, reproduced identically across two runs):
- truncated:     both-accept 0,   both-reject 256, native-accept/FFI-reject 0
- concat-gzip:   both-accept 136, both-reject 120, native-accept/FFI-reject 0
- random-stream: both-accept 2,   both-reject 254, native-accept/FFI-reject 0

No both-accept-but-disagree (no silent corruption) anywhere.
