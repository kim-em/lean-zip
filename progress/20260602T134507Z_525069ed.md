# Session 525069ed — feature #2483 (size-then-emit, Track D)

**UTC:** 2026-06-02T13:45Z · **Type:** feature (one-shot `/once`)

## Issue

#2483 — *perf: size-then-emit-winner — emit only the smallest DEFLATE block
(Track D, prng compress ~2.3×).* At level 6 `deflateRaw` emitted **three** full
blocks (stored + fixed + dynamic) and kept the smallest; on incompressible
input the stored block wins, so both expensive Huffman symbol emissions were
computed and thrown away.

## What landed

Size every level-≥5 candidate from the already-counted `tokenFreqs` and emit
only the winner.

- **`BitWriter.bitLength`** — `data.size * 8 + bitCount`, for measuring the
  dynamic tree-header bit count without emitting symbols.
- **Size helpers** in `Zip/Native/DeflateDynamic.lean`: `symbolBitCount`
  (the `Σ freq·(codeLen + extraBits)` dot product, O(#symbols)), `fixedBlockBytes`,
  `dynBlockBytes` (header bits via `writeDynamicHeader` into an empty writer),
  `dynamicCodeLengths`. Marked `@[irreducible]` so `split` doesn't unfold the
  286-element fold past `maxRecDepth`.
- **`deflateCompressed`/`deflateRaw`** level-≥5 rewritten to compare byte sizes
  and emit only the argmin (same strict-`<` tie-break as `pickSmaller`).
- **`deflateDynamicBlockCore`** — `deflateDynamicBlock` split so the dynamic
  emit reuses the sized `litLens`/`distLens` instead of recomputing
  `computeCodeLengths`. Removes the small-input fixed-cost regression the first
  cut introduced.

## Results

- **prng 1 MiB compress 16.7 → 38.8 MB/s (2.32×)** (target ~38 MB/s met); 256 KiB
  2.19×, 64 KiB 1.99×.
- Compressible patterns also improve: 1 MiB text 99.8 → 109.3 MB/s (1.09×),
  cyclic 1.04×. No regression ≥ 4 KiB (1 KiB text 0.95×, within noise).
- **Output byte-identical** — every native `out_size` unchanged across all rows.

## Proof approach (key insight)

The roundtrip/pad theorems hold for **whichever** block is chosen, so **no
size-correctness theorem is needed** — the size helpers are opaque cost models.
Byte-identity is an empirical invariant, pinned by the new `ZipTest/SizeHelpers`
conformance tests (`fixedBlockBytes`/`dynBlockBytes` == emitted `.size`; new
`deflateRaw(6)` == old emit-all reference). The 6 `deflate{Compressed,Raw}_*`
proofs `split` the new size conditions; `deflateDynamicBlockCore … =
deflateDynamicBlock data tokens` definitionally, so the dynamic lemmas transfer.
`deflateDynamicBlock_spec` unchanged (its `show` reframe is defeq through Core).

Two friction points discovered:
- `split` can't see through `let`-bound size locals → expose with `dsimp only []`
  (not `simp only []`, which mangles block terms).
- `split`ting the nested level-≥5 condition forces a deep whnf → needs
  `set_option maxRecDepth 8000` on the three `deflateRaw_*` theorems.
