## Observations

- `validLengths_single` was previously dead code (defined but never used); now
  used in `computeCodeLengths_valid`, which had duplicated its logic inline
- The `let nz` binding pattern creates a tension between readability and
  proof maintenance: hypotheses about `freqs.filter ...` need explicit
  coercion (`have hs_nz' : p ∈ nz := hs_nz`) for syntactic rewrites to work
- `kraftSum_init` and `kraftSum_append` express general foldl properties
  (accumulator associativity and append distributivity) that could be
  upstreamed, but they're specific enough to `kraftSum` that generalization
  would need a concrete use case
