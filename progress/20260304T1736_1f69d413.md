# Progress: Cross-block state support in executeSequences

- **Date**: 2026-03-04T17:36 UTC
- **Session type**: feature
- **Issue**: #559

## What was accomplished

- Added `windowPrefix` and `offsetHistory` parameters to `executeSequences` in
  `Zip/Native/ZstdFrame.lean` for multi-block Zstd frame support (RFC 8878 §3.1.1.5)
- Changed return type from `Except String ByteArray` to
  `Except String (ByteArray × Array Nat)` to thread offset history across blocks
- Match offsets can now reach into previous blocks' output via the prefix buffer
- Returned output excludes prefix bytes (only new block data)
- Default parameter values preserve single-block semantics
- Updated 5 existing test match arms for new return type
- Added 3 new tests:
  - Offset history threading across blocks (block 1 establishes offset, block 2 reuses via repeat code)
  - Cross-block back-reference via windowPrefix (block 2 matches into block 1's data)
  - Prefix stripping (returned output excludes prefix)
- All tests pass, sorry count unchanged at 0

## Decisions

- Used default parameters rather than a separate function to keep the API unified
- Prepend-then-strip approach for prefix: simple, correct, avoids modifying copyMatch
