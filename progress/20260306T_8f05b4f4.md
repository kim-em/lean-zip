- `range_forIn_eq_list_forIn`: Converts `[:n]` range forIn to `List.range'` forIn
- `range_forIn_inv`: Range `forIn` preserves property P under yield-only steps
- `forIn_fst_size_of_heq`: Key helper that resolves the body function `f` from
  a match equation (`heq`), ensuring `f` is concrete during `hf` elaboration.
  This avoids the metavariable issue where `split` fails on unresolved body functions.

## Key technical insight

When using `have := range_forIn_inv n _ _ P hinit hf`, the body function `f`
remains a metavariable during `hf` elaboration, causing `split` to fail (can't
find `if`/`match` to split on). The fix: `forIn_fst_size_of_heq` takes the
`heq` equation as its first argument, so Lean resolves `f` from `heq` before
elaborating `hf`. This is a reusable pattern for proving properties of
desugared `for` loops in Lean 4.
