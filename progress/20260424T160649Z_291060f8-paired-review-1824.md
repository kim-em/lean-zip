### A.1 Predicate correctness — OK

The guard at
[Zip/Archive.lean:617-619](/home/kim/lean-zip/Zip/Archive.lean:617):

```lean
unless flags &&& 0x0020 == 0 do
  throw (IO.userError
    s!"zip: CD entry patched-data flag bit 5 set (flags={flags}) for {name} at CD offset {pos}; lean-zip does not support APPNOTE §4.4.4 compressed patched data")
```

`flags : UInt16` is bound at
[Zip/Archive.lean:498](/home/kim/lean-zip/Zip/Archive.lean:498) via
`Binary.readUInt16LE data (pos + 8)` — no `.toNat`, no narrowing,
no de-Morgan'd alternative. The mask `0x0020 = 0b0000_0000_0010_0000`
isolates exactly bit 5. Mask-equality form
(`flags &&& 0x0020 == 0`) matches the existing bit-3-masking
convention `dataDescriptorBitMask : UInt16 := 0xFFF7` at
[Zip/Archive.lean:23](/home/kim/lean-zip/Zip/Archive.lean:23),
rather than the logically-equivalent but stylistically-divergent
`flags &&& 0x0020 != 0x0020` form. Boundary check: `0x0000`,
`0x0800` (bit 11 UTF-8), `0x0008` (bit 3 data descriptor),
`0x0FFF ∧ ¬0x0020` all pass; `0x0020` (fixture value), `0x0021`,
`0x0820`, `0xFFFF` all reject.
