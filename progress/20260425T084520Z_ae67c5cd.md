Section structure (62 lines added; file 244 → 306 lines):

- **Headline rule** — per-slot fixtures should use a slot-distinct
  printable ASCII prefix in the smuggled value to encode slot-identity
  into `xxd` diffs; the prefix is what a parser-differential caller
  would see if `Binary.fromLatin1` truncated at the NUL.
- **Examples table** (7 rows) covering all post-#1928 wave fixtures:
  the 5 UStar interior-NUL slots (`name`/`linkname`/`prefix`/`uname`/
  `gname`, PRs #1880/#1934/#1937/#1944/#1957) plus the 2 GNU long-name
  / long-link interior-NUL fixtures (PRs #1865/#1953). Columns:
  family, slot, fixture, smuggled value, prefix, closing PR.
- **Cross-cutting rules**:
  1. `.lnk` filename-extension as a slot-identity marker for
     link-typed slots (UStar `linkname`'s `evil.lnk\x00.tar` and GNU
     `long-link`'s `safe.lnk\x00rogue.tar` — introduced by PR #1953).
  2. *trailing-NUL invariant* — last byte of smuggled value must be
     non-NUL, otherwise `stripTrailingNuls` (GNU long-name path) or
     the `findIdx? (· == 0)` interior-NUL guard's input invariant
     (UStar path) would erase the interior NUL the test intends.
- **Writer-side override hooks subsection** — `pathOverride` is the
  only override hook in `Tar.buildHeader` today (because `path` is
  auto-split via `splitPath`); `linkname`/`uname`/`gname` use the
  first-class entry fields directly because `Binary.writeString`
  copies bytes verbatim. Cites the post-#1944 paired-review §E.7.g
  and post-#1957 feature-progress *Patterns / decisions* sections
  for the rationale (early planning notes proposed parallel
  `linknameOverride` / `unameOverride` / `gnameOverride` hooks; the
  PRs #1934 / #1944 / #1957 actually shipped the simpler
  first-class-field pattern).
