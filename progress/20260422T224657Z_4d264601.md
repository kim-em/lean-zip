## Summary

Threaded the central-directory general-purpose bit flag word through
the parser into `Archive.Entry`, and added a CD/LH flags-consistency
check in `readEntryData` that fires before the compressed-payload read.
The check XOR-masks out bit 3 (the data-descriptor flag, which is a
legitimate per-segment LH concern) and rejects any mismatch on the
remaining 15 bits — closing a known ZIP-smuggling vector where a CD
with bit 11 (UTF-8 name) set and an LH with bit 11 clear would trick
downstream LH re-parsers into decoding the name under the Latin-1
fallback path while the extractor used the CD-derived strict-UTF-8
name.

## Decisions and design notes

- **Default `flags := 0`**: chosen so existing struct literals in the
  test code (and future writer-side callers) stay green. The writer
  side never reads `entry.flags` — `writeLocalHeader` and
  `writeCentralHeader` both emit the literal `0x0800` at offsets 6 and
  8 respectively. The new `readEntryData` check on a writer-generated
  `Entry` sees `entry.flags = 0` and `localFlags = 0x0800`; the
  `flagsMask = 0xFFF7` XOR-masks bit 3 only, so bits 11 on CD vs LH
  must match — for writer-round-trip archives both read as `0x0800`
  from the CD (via `parseCentralDir`) and the LH is also `0x0800`, so
  the check passes without writer-side changes.  No regression in the
  writer-round-trip tests.

- **`0xFFF7` mask**: all 16 bits except bit 3. Matches the PKZIP
  APPNOTE semantics — bit 3 (data-descriptor present) legitimately
  differs between CD and LH because the CD always carries the resolved
  crc/size values while the LH zeroes them when bit 3 is set.  The
  existing `usesDataDescriptor` branch at line 604 already relies on
  this asymmetry.

- **Scope**: intentionally scoped to the XOR-mask comparison.
  Per-bit semantics (encryption bit 0, strong-encryption bit 6,
  UTF-8 bit 11) each warrant their own check and its own error message;
  this PR's task is only the mask-level consistency check so that any
  mismatch is rejected before the payload read.  Follow-up per-bit
  checks can layer on top without re-opening this one.
