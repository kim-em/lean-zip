## Decisions / patterns

- **`pack_unpack` vs `unpack_pack_of_valid`**: initially tried a single
  `Valid`-based theorem but the native proof's nil-prefix branch
  (`a = ByteArray.empty`) requires `pack (unpack init) = init` for an
  *arbitrary* `init : UInt32` (the state `unpack init` may not be
  Valid). Proving both inverse directions and using the right one
  per-case was cleaner than the alternatives suggested in the issue
  (adapted lemma specialised to cons-chains, or two Native theorems).
- **Refactored the bitwise computation into a private helper.**
  The toNat computation for `pack` ‚Äî reducing `(ofNat b <<< 16) ||| ofNat a`
  to `a + b * 2^16` via `Nat.two_pow_add_eq_or_of_lt` ‚Äî is the same
  calculation used in both `pack_unpack` and `unpack_pack_of_valid`.
  Extracting `pack_toNat_of_bounds` turned ~30 lines of duplicated
  `simp`/`rw`/`omega` into one `simp only [pack_toNat_of_bounds _ _]`
  call per use site.
- **`Nat.two_pow_add_eq_or_of_lt` is the key lemma.** Replaces the
  bitwise OR of non-overlapping bits with plain addition; `omega`
  takes over from there. Aligns with how CRC-32 (landing as #1635)
  uses `bv_decide` for its xor-cancellation lemma ‚Äî both reduce the
  "collapse a bitwise algebraic identity" step to a single automation
  call in their respective regimes.
- **No `Std.Tactic.BVDecide` import needed.** Unlike the CRC-32
  sibling, Adler-32's pack/unpack identity is natively a Nat
  arithmetic statement (mod/div 65536) that `omega` handles directly
  once `Nat.two_pow_add_eq_or_of_lt` has eliminated the `|||`.
