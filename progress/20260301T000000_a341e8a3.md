### Bare simp reductions (15 conversions)

Converted 15 bare `simp` calls to more precise alternatives:

- **`simp only` + `; contradiction`** (6 instances): For `| none =>` cases where
  `simp` was used to derive `none = some (...)` and close the goal. Replaced with
  explicit `simp only [hypothesis] at h; contradiction`.

  `none` impossibility.

### Bare simp remaining (~45)

The remaining bare `simp` calls fall into categories that resist `simp only`:

1. **Cascade Option.bind patterns** (~18 in `decodeLitLen_ge257_isReference` and
   `decodeLitLen_reference_inv`): Deeply nested match/bind chains from unfolded
   `decodeLitLen`. `simp only` hits `maxRecDepth` because each rewrite exposes
   more `Option.bind` nodes.
2. **UInt16/Nat conversions** (~10): `simp [Nat.toUInt16, ...]`, `simp [beq_iff_eq]`,
   etc. Exact lemma names for `simp only` are fragile and version-dependent.
3. **Table size proofs** (~8): `simp [Deflate.Spec.lengthBase]; omega` where simp
   unfolds the array literal to compute `.size`.
4. **Other** (~7): `simp_all`, `simp at this; omega`, `by simp` for ByteArray lemmas.

## Decisions

- Kept `set_option` directives — genuinely needed for proof depth
- Used `; contradiction` pattern instead of relying on `simp` to close `none = some`
  goals automatically
