# Progress: End-to-end decompressZstd composed completeness for single compressed blocks

- **Date**: 2026-03-12T03:45Z
- **Session**: 21e9cfaa (feature)
- **Issue**: #1186

## Accomplished

Added two capstone theorems in `Zip/Spec/ZstdFrame.lean` composing the full
completeness chain from byte-level conditions to `decompressZstd` success
for compressed blocks:

1. **`decompressZstd_succeeds_single_compressed_zero_seq_frame`** — byte-level
   frame header + compressed block (type=2, last=1) + literals/sequences parsing
   → `∃ output, decompressZstd data = .ok output`

2. **`decompressZstd_succeeds_single_compressed_sequences_frame`** — same for
   compressed blocks with numSeq > 0, including FSE table resolution, backward
   bitstream init, sequence decoding, and execution

Each composes the chain:
- `parseFrameHeader_succeeds` (byte-level → frame header parsed)
- `decompressFrame_succeeds_single_compressed_zero_seq/sequences` (header + block → frame)
- `decompressZstd_single_frame` (frame + end-of-data → API success)

## Decisions

- **Universally quantified hypotheses**: Header field constraints and block
  conditions are quantified over `(hdr, afterHdr)` from `parseFrameHeader`,
  matching the pattern established by #1175 (raw/RLE composed completeness).

- **Combined `hparsing` hypothesis**: For the zero-seq case, literals parsing
  and sequences header are combined into one universally quantified hypothesis
  with existentials. For the sequences case, the full pipeline (literals →
  sequences header → FSE tables → bitstream init → decode → execute) is
  combined similarly.

- **`hterm` hypothesis**: Frame-fills-data condition stated as
  `∀ content pos', decompressFrame ... = .ok (content, pos') → pos' ≥ data.size`
  rather than byte-level, matching #1175's approach.

- **Placement**: Added in a new `/-! ## decompressZstd composed completeness for
  compressed blocks -/` section at the end of `ZstdFrame.lean`, before `end`.
