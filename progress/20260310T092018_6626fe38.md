## Accomplished

Added three parsing completeness theorems to `Zip/Spec/Zstd.lean`:

1. **`parseBlockHeader_succeeds`**: When data has ≥3 bytes from `pos` and the
   block type field is not reserved (3), `parseBlockHeader` returns `.ok`.
   Proof uses `split` on the match + `bv_decide` to eliminate the catch-all
   case (showing `expr &&& 3` can only be 0-3).

2. **`decompressRawBlock_succeeds`**: When data has ≥`blockSize.toNat` bytes
   from `pos`, `decompressRawBlock` returns `.ok`. Simple unfold + guard
   elimination.

3. **`decompressRLEBlock_succeeds`**: When data has ≥1 byte from `pos`,
   `decompressRLEBlock` returns `.ok`. Same pattern as raw block.

These are the first **completeness** theorems in the Zstd spec layer.
Existing theorems only cover the correctness direction ("if parsing
succeeds, then..."). These new theorems cover the converse ("if input is
well-formed, then parsing succeeds").

## Key decisions

- Used `bv_decide` (SAT-based BitVec decision procedure) to close the
  `parseBlockHeader` catch-all case. `omega` and `bv_omega` cannot reason
  about bitwise AND; `bv_decide` handles it by treating opaque UInt8
  sub-expressions as universally quantified bitvector variables.
- Placed theorems in a new `/-! ## Parsing completeness -/` section between
  existing block-level correctness theorems and frame-level theorems.
