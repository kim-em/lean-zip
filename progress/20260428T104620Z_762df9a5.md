## Accomplished

Wired the `miniz_oxide` v0.8 raw-DEFLATE Rust crate as the third Track D
comparator (alongside zlib and the not-yet-landed libdeflate / zopfli),
end-to-end:

- `rust/miniz_oxide_shim/` — `staticlib` Cargo crate exposing a tiny
  C-ABI surface (`lean_miniz_oxide_compress` / `_decompress` / `_free`).
  `Cargo.toml` pins `miniz_oxide = "0.8"`, `panic = "abort"` so we never
  unwind across the FFI boundary, `lto = "thin"` for release.
- `c/miniz_oxide_ffi.c` — Lean FFI shim. Two modes: with
  `-DHAVE_MINIZ_OXIDE` it forwards to the Rust functions and copies the
  result into a Lean `ByteArray`; without it (the fail-soft mode) all
  entry points return `IO.userError "miniz_oxide: not built with Rust
  support …"`.
- `Zip/MinizOxide.lean` — `MinizOxide.compress`,
  `MinizOxide.decompress` opaque externs. `compress`'s `level` argument
  matches the rest of the bench (default 6); `decompress`'s
  `maxDecompressedSize` defaults to 1 GiB and shares the
  `"exceeds limit"` error wording with `RawDeflate.decompress`.
- `lakefile.lean` — `minizOxideEnabled` predicate (env knobs:
  `MINIZ_OXIDE_DISABLE`, `MINIZ_OXIDE_LDFLAGS`; otherwise `cargo
  --version` autodetect). When enabled, `linkFlags` shells out to
  `cargo build --release` and appends `-L… -lminiz_oxide_shim`. The C
  shim's `-DHAVE_MINIZ_OXIDE` consults the *same* predicate so a
  `MINIZ_OXIDE_DISABLE=1` rebuild remains internally consistent.
- `shell.nix` — `pkgs.cargo` + `pkgs.rustc` so NixOS dev/CI gets the
  comparator out of the box.
- `ZipBench.lean` — adds `compress-miniz` and `inflate-miniz` ops, plus
  a `text` pattern (mirrors `ZipTest.Helpers.mkTextData`) so the
  verification-command line `lake exe bench compress-miniz 65536 text 6`
  works.
- `ZipTest/MinizOxide.lean` — roundtrip + cross-decoder (miniz↔zlib) +
  decompression-cap smoke tests. When the disabled-marker
  `"miniz_oxide: not built with Rust support"` shows up the suite
  prints a skip line and returns `Unit`, so `lake exe test` keeps
  passing on minimal toolchains.
- `BENCH.md` (new) + `README.md` blurb — comparator-toolchain matrix,
  env-knob reference, per-platform verified status table.

## Decisions / patterns

- **Fail-soft via `-DHAVE_MINIZ_OXIDE`, not conditional Lake
  registration.** Conditionally registering `extern_lib` /
  `lean_lib` based on cargo availability would have required either
  conditional imports (which Lean doesn't do) or a separate optional
  module that's not pulled in by `Zip.lean`. Compiling the C shim with
  a runtime stub keeps `import Zip.MinizOxide` valid in all
  configurations and pushes the failure mode to the runtime where the
  smoke tests can match it as a substring and skip cleanly.
- **One predicate for both link flags and CFLAGS.** First attempt had
  `-DHAVE_MINIZ_OXIDE` driven by `minizRustLib.pathExists` — that
  disagreed with `linkFlags`'s `minizOxideEnabled` check during a
  `MINIZ_OXIDE_DISABLE=1` rebuild and produced an `undefined symbol`
  link error. Unifying around `minizOxideEnabled` eliminated the class.
- **No equivalence proofs.** miniz_oxide is unverified Rust; this is a
  comparator only. Smoke tests assert ecosystem agreement (miniz output
  decompresses with zlib, and vice versa), not specification
  correctness.
