# Review: BitReaderInvariant + BitWriterCorrect proof quality audit

**Date**: 2026-03-12 UTC
**Session type**: review
**Issue**: #1303

## Summary

Audited `Zip/Spec/BitReaderInvariant.lean` (655 lines, 30 theorems) and
`Zip/Spec/BitWriterCorrect.lean` (412 lines, 28 theorems) for proof quality.

## Key finding

The issue estimated ~153 bare `simp` calls across both files. In fact, **all
`simp` calls are already `simp only [...]`** — the grep pattern `simp[^_]`
used to estimate the count also matches `simp only`. Zero bare simps existed.

## Changes made

### BitReaderInvariant.lean
- `alignToByte_pos_le`: replaced ambiguous `simp only; omega` with
  `dsimp only; omega` (clarifies intent: reduce struct projections before omega)
- `decodeStored_inv`: combined `rw [hN] at h; simp only [] at h` into
  `simp only [hN] at h` (2 occurrences — eliminates redundant empty simp passes)

### BitWriterCorrect.lean
- `beq_mod2_eq_bne`: replaced 6-line case-split proof with one-liner
  `decide_mod2_eq_bne x` (for Nat, `BEq.beq` is definitionally `decide`)
- Replaced 6 bare `simp only` (no lemma list) with `dsimp only` throughout
  `addBit_toBits`, `addBit_wf`, `uint16_extract_bit`, `uint32_extract_bit`,
  `writeHuffCode_go_spec`, `writeBits_go_spec` — these calls only expand
  let-bindings and reduce struct projections, so `dsimp only` is more precise

## Observations (no changes needed)

- **readUInt16LE_inv / readBytes_inv**: Near-identical 17-line proofs, but
  extracting a shared helper would be over-engineering for private helpers
- **uint16_extract_bit / uint32_extract_bit**: Structurally identical, but
  abstracting over UInt types would be complex and not worth 20 lines saved
- **decodeCLSymbols_inv sym==16/17/18 branches**: Repetitive but structurally
  necessary — each branch has slightly different split structure
- All `simp only []` calls after `obtain` destructuring are legitimate
  (normalizing after product destruction)
- File organization is clean: individual property lemmas → bundled `_inv`
  lemmas → public API lemmas used by InflateLoopBounds and InflateComplete
