## Decisions and patterns

- **Row count is 29, not 23 as the issue body estimated.** The
  verification command
  `ls testdata/zip/malformed testdata/tar/malformed testdata/tar/security | grep -E '\.(zip|tar)$' | wc -l`
  returns 29 on today's tree. The table has 29 rows to match —
  the issue's "today: 23" figure was stale.
- **First-landed column mixes PR numbers and a commit hash.**
  Fixtures added via a dedicated PR use the PR number (#1497,
  #1543, #1544, #1545, #1546, #1554, #1555). Fixtures inherited
  from the initial `lean-zlib → lean-zip` rename commit use the
  short SHA `481e562` — that commit was not landed via a numbered
  PR (confirmed via `gh api repos/.../commits/.../pulls`, which
  returned an empty list). The preamble documents this convention.
- **`backslash-slip.tar` annotated for the orphan-then-assertion
  case.** The issue called out that this fixture was orphaned
  until PR #1555 added an assertion. The *First landed* cell
  therefore reads `481e562 (assertion added by #1555)` to make the
  history visible without needing a separate "Missing work"
  sub-bullet.
- **No "Missing work" bullet needed.** Every one of the 29
  fixtures has a live assertion under `ZipTest/`
  (`ZipFixtures.lean`, `TarFixtures.lean`, or — for
  `invalid-utf8-with-flag.zip` only — `Utf8Fixtures.lean`). I
  grep'd each filename across `ZipTest/` before writing each row;
  the preamble states this explicitly.
- **`hardlink-outside.tar` is a positive regression.** It does not
  throw; the assertion is that the extract directory remains
  empty. The preamble calls this out so an auditor scanning the
  "Defence exercised" column doesn't expect a thrown error.
