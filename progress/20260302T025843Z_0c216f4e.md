## What was accomplished

- Created `Zip/Native/ZstdFrame.lean` (~100 lines):
  - `ZstdFrameHeader` structure with all RFC 8878 §3.1.1 fields
  - `windowSizeFromDescriptor` helper for computing window size from
    exponent/mantissa encoding
  - `parseFrameHeader` validates magic number (0xFD2FB528), parses
    frame header descriptor bit fields, optional window descriptor,
    optional dictionary ID (0/1/2/4 bytes), and optional frame content
    size (0/1/2/4/8 bytes with +256 offset for 2-byte case)
  - Uses existing `Binary.readUInt16LE`/`readUInt32LE`/`readUInt64LE`
    utilities

- Created `ZipTest/ZstdNative.lean` (~70 lines):
  - 9 tests covering empty, small (62B), medium (64KB), and large (124KB) inputs
  - Tests at compression levels 1, 3, and 19
  - Verifies content size matches input size
  - Validates header end position is within bounds
  - Tests error handling: invalid magic number, truncated input

- Registered module in `Zip.lean` and `ZipTest.lean`
