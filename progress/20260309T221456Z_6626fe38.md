### 1. Content pipeline is mechanical but undocumented

The block→frame→API lifting pattern has been applied 20+ times with
consistent structure. Each layer has a fixed proof skeleton:
- Block-level: WF unfolding + state threading (hard)
- Frame-level: unfold `decompressFrame` + plug block theorem (mechanical)
- API-level: apply single-frame lifting theorem (mechanical)

**Action**: Added "Content Pipeline: Block → Frame → API Lifting"
section to `lean-zstd-spec-pattern` skill with templates for each layer.

`Zip/Spec/Zstd.lean` has grown to **2827 lines** — 2.8× the 1000-line
threshold from `review.md`. This is the root cause of the persistent
merge conflict problem. Every content pipeline PR touches this file.

**Action**: Updated hot file tracking in `agent-pr-recovery` skill.

→ `ZstdContent.lean`. A planner should create a file-split issue with

### 3. Helper extraction heuristics emerged

Across 20+ sessions, a consistent pattern: extract at ≥3 call sites,
or ≥2 with ≥6-line savings. Sessions that over-extracted caused
maintenance issues; sessions that under-extracted had redundant proofs.

**Action**: Added extraction heuristic to `proof-review-checklist` skill.
