## What was accomplished

Bundled all three PR #1607 recommended-change findings into a single
~20-line diff on `ZipTest/FuzzInflate.lean`:

1. **Smoke `runFuzz` N bumped 100 → 1000** (§2 recommendation).
   At the review's measured ~100 μs/iteration, this adds ~100 ms to
   `lake exe test`. Empirically: full test run measured 8.33 s wall
   clock end-to-end on this worktree, well inside the "fast" range.

2. **Seed printed in the smoke output line** (§2 recommendation).
   The new header line now reads
   `  FuzzInflate tests (seed=0xdeadbeef)...` — verified by running
   `lake exe test` locally. Format uses
   `String.ofList (Nat.toDigits 16 seed.toNat)` to produce a lowercase
   hex string with no leading zero padding; `0xdeadbeef` reads
   naturally. (First pass used the deprecated `.asString` method on
   `List Char`; swapped to `String.ofList` after the build flagged the
   deprecation warning.)

3. **`tryFFI` narrowed to `.userError` only** (§3 recommendation).
   The broad `catch e => ...` was replaced with
   `catch e => match e with | .userError _ => ... | _ => throw e`.
   A new docstring paragraph records why the narrowing is safe
   today (zlib FFI's only `IO.Error` variant is `.userError` via
   `lean_mk_io_user_error` in `c/zlib_ffi.c`) and what the re-raise
   protects against (future FFI contract regressions that add new
   variants; runtime-level errors like `.unexpectedEof` that should
   surface loudly). `tryNative` and `tryStreaming` were deliberately
   left untouched per the issue's scope-discipline list.

No other files touched. `ZipFuzzInflate.lean`,
`scripts/fuzz-inflate.sh`, `chunkSizes`, `sizeClasses`, and all
`Zip/**` sources are unchanged.

## Decisions made

- **Hex over decimal for the seed print.** The existing wall-clock
  exe `[fuzz-inflate]` header prints `seed={seed}` in decimal
  (UInt64 `toString`), but the issue's body explicitly recommended
  hex and the default `0xdeadbeef` seed reads naturally in hex.
  Picked hex. If a reviewer prefers decimal for exe/smoke
  consistency, a one-line flip to `s!"seed={seed}"` is trivial.

- **`String.ofList` over deprecated `List.asString`.** First build
  surfaced a deprecation warning; swapped. Keeps the warning count
  at zero.

- **Left `tryNative` and `tryStreaming` untouched.** Issue's
  scope-discipline list explicitly rules these out. `tryNative`
  takes `Except String ByteArray` anyway (no `IO.Error`); widening
  the narrowing to `tryStreaming` would be a separate defensible
  edit but a future planner's call.
