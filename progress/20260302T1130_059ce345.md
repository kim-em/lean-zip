# Progress: Track E — Interleaved FSE sequence decode loop

- **Date**: 2026-03-02T11:30 UTC
- **Session type**: feature
- **Issue**: #469

## Accomplished

1. **Implemented `decodeSequences`** in `Zip/Native/ZstdFrame.lean` (~76 lines):
   - Takes three FSE tables (litLen, offset, matchLen), a BackwardBitReader, and numSeq
   - Initializes three FSE states by reading `accuracyLog` bits each
   - Loops numSeq times with correct RFC 8878 §4.1.1 interleaving:
     - Symbol lookup order: offset, matchLen, litLen
     - Extra bits reading order: offset, matchLen, litLen
     - State update order (not on last seq): litLen, matchLen, offset
   - Returns `Array ZstdSequence` ready for `executeSequences`

2. **Added 4 interleaved decode tests** in `ZipTest/ZstdNative.lean` (~152 lines):
   - Test 84: single-sequence decode with minimal 2-cell FSE tables
   - Test 85: two-sequence decode verifying state update order
   - Test 86: zero sequences returns empty array
   - Test 87: codes with extra bits (litLen code 16, matchLen code 32, offset code 3)

3. **Derived `Inhabited` for `ZstdSequence`** to support array indexing in tests.
