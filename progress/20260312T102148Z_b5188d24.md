Both follow the established frame-level lifting pattern: call the block-level
theorem, unfold `decompressFrame`, thread through dictionary/checksum/size guards.

## Key Decision

The block-level `executeSequences` hypothesis uses a windowed-output expression
`(if windowSize > 0 && output.size > windowSize.toNat then output.extract ... else output)`.
At the frame level, output starts as `ByteArray.empty`, so this always reduces
to `ByteArray.empty`. Rather than exposing the verbose `if` expression in the
frame-level API, I kept the hypothesis clean (`ByteArray.empty`) and added a
`have h_win ... ; rw [← h_win] at hexec1` conversion step in each proof.
