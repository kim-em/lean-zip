## Accomplished

1. **`decodeFourHuffmanStreamsWF`** — WF variant of the 4-stream Huffman literal
   decoder using `decodeHuffmanStreamWF` (structural recursion on count) instead
   of `decodeHuffmanStream` (opaque `for` loop). Placed after original in
   `Zip/Native/ZstdHuffman.lean`. Original kept intact.

2. **`decodeFourHuffmanStreamsWF_size`** — When the WF variant succeeds, the
   output has exactly `regenSize` bytes. Proved by:
   - Single-pass `simp only` to unfold do-notation
   - `iterate 11` batch elimination of guard/bind error branches
   - `rename_i` to access the 4 decode hypotheses
   - `decodeHuffmanStreamWF_size` applied to each stream
   - `omega` for the final arithmetic

## Decisions

- **Added precondition `3 * ((regenSize + 3) / 4) ≤ regenSize`**: The plan
  claimed this identity holds for all `regenSize`, but it fails for
  `regenSize ∈ {1, 2, 5}` due to Nat subtraction underflow. The precondition
  is always satisfied in practice (Zstd 4-stream mode requires `regenSize ≥ 256`).
