## Decisions

- **Guard placement: after `name` decode.** Preferred the
  operator-legible "with-name" variant per the issue's placement note,
  matching the shape of the PR #1801 method-allowlist guard and
  PR #1773 stored-method guard. The version-check fires *before* the
  method allowlist so the two guards stay deterministically ordered
  — useful for error attribution in fixtures that violate both
  invariants.
- **Half-closed two-step for `SECURITY_INVENTORY.md` *Recent wins*.**
  Deferred rather than inlined: the PR number is only known after
  GitHub assigns it, and inlining a placeholder would trip the
  inventory linter's placeholder-PR rule introduced in PR #1796. The
  line-anchor shifts are included in the same commit because the
  linter is strict and the shifts are mechanical.
- **Fixture size/shape.** Reused the stock `write()` helper from
  `scripts/build-cd-lh-mismatch.py` with `lh_method=0, cd_method=0,
  lh_version=51, cd_version=51`. The `P`-byte payload path is fine:
  the new guard fires at CD parse time, before any LH or payload
  read, so the 6-byte `hello\n` bytes are never interpreted.
  Deviation from the issue's suggested `crc=0, compSize=0,
  uncompSize=0` shape is intentional — using the existing write()
  defaults keeps the builder clean, and the early-reject semantics
  make the sizes irrelevant.
