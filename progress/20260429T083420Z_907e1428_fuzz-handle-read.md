## What was accomplished

1. **`ZipTest/FuzzHandleRead.lean`** — deterministic xorshift-seeded
   fuzz driver mirroring `ZipTest/FuzzInflate.lean`'s shape but
   targeting the `Handle.read` / `Stream.read` archive-parser
   surface. Drives `Archive.list`, `Archive.extract`, `Tar.list`,
   `Tar.extract`, `Tar.extractTarGz`, `Tar.extractTarGzNative`,
   `Gzip.decompressStream`, `Gzip.decompressFile`, and
   `RawDeflate.decompressStream` with random bytes routed through
   both the file `Handle.read` path (via temp fixtures under
   `/tmp/lean-zip-fuzz-handle-read/input.bin`) and the
   `IO.FS.Stream.read` path (via `byteArrayReadStream` +
   `fragmentingStream` from `ZipTest/Helpers.lean`).

2. **Pathological-input families** (per #2380 deliverable 3):

   - `sizeClasses = #[0, 1, 16, 512, 8192, 65536, 131072]` —
     `0` exercises the empty-input EOF path; the upper bound
     (128 KiB) is deliberately above the 65536-byte chunk size used
     by every streaming surface so cross-chunk realloc transitions
     are exercised on every iteration.
   - `chunkSizes = #[1, 7, 31, 127, 65535, 65536, 65537]` —
     values straddling 65536 stress the realloc-grown buffer
     transitions inside the streaming inflate / tar parsers.
   - `outputCapClasses = #[0, 64, 1024, 65536, 1024 * 1024]` —
     the unlimited-mode (`0`) and finite-cap branches of the
     `maxDecompressedSize` / `maxEntrySize` / `maxTotalSize`
     parameters.
   - `withEocdSignature` — splices the ZIP EOCD signature
     (`PK\x05\x06`) into the last 64 KiB of random bytes so
     `Archive.list` proceeds past the tail scan into the
     central-directory read paths (where the actual `Handle.read`
     drivers live).
   - `withGzipMagic` — stamps the gzip header (`0x1f 0x8b 0x08
     0x00 …`) so `Gzip.decompressFile` / `Tar.extractTarGz`
     commit to the streaming-inflate read loop.
   - `withUstarMagic` — stamps `"ustar\x00"` + `"00"` at offset
     257/263 so tar `parseHeader` attempts a full UStar parse
     instead of bailing on magic mismatch.

3. **`ZipFuzzHandleRead.lean`** + **`scripts/fuzz-handle-read.sh`** —
   sibling top-level driver + wrapper script analogous to
   `ZipFuzzInflate.lean` + `scripts/fuzz-inflate.sh`. Wall-clock
   budget read from `LEAN_ZIP_FUZZ_HANDLE_READ_SECONDS`

   `LEAN_ZIP_FUZZ_HANDLE_READ_SEED` (default

4. **`lakefile.lean`** — registered `lean_exe fuzz_handle_read`

5. **`ZipTest.lean`** — wired `import ZipTest.FuzzHandleRead` and
