## Accomplished

Implemented Huffman tree descriptor decoding for the direct representation
mode (RFC 8878 §4.2.1), which handles Zstd files where the compressor
packs symbol weights as 4-bit values rather than FSE-compressing them.

### New structures
- `HuffmanEntry`: symbol + numBits for flat table lookup
- `ZstdHuffmanTable`: maxBits + flat table array

### New functions
- `parseHuffmanWeightsDirect`: unpack 4-bit weights (2 per byte) from
  packed byte representation
- `weightsToMaxBits`: derive maxBits from explicit weight array,
  accounting for the implicit last symbol
- `buildZstdHuffmanTable`: convert weights to flat lookup table with
  correct code assignment and entry filling
- `parseHuffmanTreeDescriptor`: read header byte, dispatch direct mode
  (< 128) or return error for FSE mode (>= 128)

### Tests (10 new, tests 45–54)
- Weight unpacking from known byte sequences
- maxBits derivation for 2-symbol and 4-symbol cases
- Table construction verification (entry count, numBits, symbol mapping)
- Full descriptor parsing (direct mode)
- Error cases: FSE mode, truncated input, all-zero weights

## Decisions
- Used flat lookup table (array of size 2^maxBits) matching existing FSE
  table pattern. Each entry stores symbol + numBits for O(1) decoding.
- Trim trailing zero weights from packed representation, since packed bytes
  may pad with a zero nibble for odd symbol counts.
- Weight-to-code assignment: ascending weight order, sequential codes,
  with entries filling 2^(W-1) slots per symbol.
