**PR**: #1409

## What was done

Extracted the L1 (base predicates + correctness theorems) and L2 (block-loop
structural lemmas) sections from `Zip/Spec/Zstd.lean` into two new files:

- `Zip/Spec/ZstdBase.lean` (528 lines) — magic number predicates, frame/block
  header predicates, DecidableEq instances, correctness theorems for
  parseBlockHeader/decompressRawBlock/decompressRLEBlock, parsing completeness,
  field characterization, frame-level output guarantees, skippable frame spec
- `Zip/Spec/ZstdBlockLoop.lean` (493 lines) — decompressBlocksWF structural
  properties (output_size_ge, prefix, pos_gt, le_size), single-block content
  (single_raw, single_rle), composed completeness (succeeds_single_raw/rle),
  frame-level composed completeness (decompressFrame_succeeds_single_raw/rle)

`Zip/Spec/Zstd.lean` went from 6974 to 6023 lines (~951 lines removed).

## Decisions

- The original issue proposed `ZstdBlockLoop` importing `Zstd` with a reverse
  import, which creates a circular dependency. Instead, also extracted L1 into
  `ZstdBase` to create a clean DAG: `ZstdBase <- ZstdBlockLoop <- Zstd`.
- Local macros (`unfold_except`, `frame_from_blocks`) duplicated in each file
  since Lean's `local macro` is file-scoped.
