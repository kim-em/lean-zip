## What was accomplished

### 1. Combined duplicate loop induction proofs

Merged `executeSequences_loop_inv` and `executeSequences_loop_output_size_ge`
which shared identical 6-way case analysis structure. The combined
`executeSequences_loop_inv` now proves all four properties in a single
induction pass:
- Output size equation (size = initial + literals + match bytes)
- litPos lower bound (litPos ≤ litPos')
- litPos upper bound (litPos' ≤ literals.size)
- Output monotonicity (output'.size ≥ output.size)

`executeSequences_loop_output_size_ge` is now a short corollary that
destructures the result tuple, derives `hlp` from success in the cons
case, and extracts the 4th conjunct.

### 2. Bare `simp` elimination

Replaced bare `simp` calls with targeted alternatives:
- `simp [copyBytes]` → `simp only [copyBytes, ↓reduceIte]` (+ `omega`)
- `simp [ByteArray.size_push]` → `simp only [ByteArray.size_push]; omega`
- `simp at h` (discrimination) → `exact nomatch h` (4 instances)
- `simp` (foldl nil) → `simp only [List.foldl_nil, Nat.add_zero]`

**Remaining bare `simp`**: 6 instances of `<;> simp <;> omega` in
`resolveOffset_history_valid_*` proofs. These reduce `Array.getElem!`
on literal arrays (e.g. `#[a, b, c][0]! > 0`). Neither `simp only []`
nor `dsimp only` can handle this reduction — the full simp set is
needed. These are documented and acceptable.

### 3. Naming consistency audit

All naming conventions are consistent:
- `resolveOffset_*`: clear pattern with `_positive_*`, `_repeat{1,2,3}_val`,
  `_shifted{1,2,3}_val`, `_history_valid_*`, `_history_size`
- `copyBytes_*`/`copyMatch_*`: `_size`, `_getElem_lt`, `_getElem_ge`
- `_ge`/`_lt`/`_gt` suffixes match mathematical meaning
- `_hsize` params prefixed with `_` serve as documentation preconditions

No renames needed.
