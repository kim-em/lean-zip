- **Date**: 2026-03-13 UTC

## Accomplished

Converted 4 of 5 `[i]!` (runtime bounds-checked) array accesses in
`decodeHuffman.go` to proven-bounds `[i]` access:

1. `lengthBase[idx]` — bounds from `if h : idx ≥ lengthBase.size`
2. `lengthExtra[idx]` — bounds via `lengthExtra_size` + `lengthBase_size`
3. `distBase[dIdx]` — bounds from `if h : dIdx ≥ distBase.size`
4. `distExtra[dIdx]` — bounds via `distExtra_size` + `distBase_size`

Added `@[simp]` size lemmas for all 5 constant tables.

Fixed proof breakage in 3 spec files:
- **DecodeCorrect.lean**: bridged proven-bounds ↔ `getElem!` using
  `getElem!_pos` for `readBits_toBits` calls and copyLoop correspondence
- **DecodeComplete.lean**: changed `↓reduceIte` → `↓reduceDIte` for dite
  branches, added `simp only [← getElem!_pos]` to normalize goals
- **InflateRawSuffix.lean**: `if_neg` → `dif_neg`, `← getElem!_pos` bridging

## Key patterns discovered

- `simp only [← getElem!_pos]` converts proven-bounds back to `!` form in
  goals, allowing existing `!`-based hypotheses to apply without modification
- When `ite` → `dite` in function body, proofs need `↓reduceDIte` instead of
  `↓reduceIte`, and `dif_neg` instead of `if_neg`
- A single `simp only [← getElem!_pos]` can normalize ALL proven-bounds
  accesses in a goal at once — no need to repeat for each table
