### Sub-section: ZIP64-specific consistency sub-layer

The `findEndOfCentralDir` ZIP64 branch now hosts **two** structurally
related checks:

| Layer            | Check                       | Throws at         | Direction              | Depends on                |
|---|---|---|---|---|
| Framing          | EOCD64 record-size          | `:319`            | equality to 44         | PR #1754's IO lift        |
| Override sentinel| 6-field override-vs-standard| `:337/340/343/346/349/352` | sentinel ∨ numeric match | PR #1754's IO lift |

Structural observations:

1. **Ordering is load-bearing**: the record-size check validates the
   *frame* (is this a well-formed v1 EOCD64 header?) before any of
   the *fields inside the frame* are compared to their standard-EOCD
   counterparts. Swapping the order would let an archive with a
   malformed record-size but sentinel-correct override fields fall
   through to the override block where the reader would trust
   per-field offsets inside a frame that is not what it claims to be.
2. **Depending on the IO lift**: both checks are only possible because
   PR #1754 lifted `findEndOfCentralDir`'s return type from
   `Option (...)` to `IO (Option (...))`. PR #1761 is the **second**
   beneficiary and the first to exercise a new error substring (PR
   #1754 used one, PR #1761 adds a second) — the lift is thus
   load-bearing for any future ZIP64-branch check (e.g. PRs
   #1755/#1764 when they rebase).
3. **Class-label continuity**: paired-review #1754 §G.2 flagged the
   `other (ZIP64 consistency)` class label as a **novel singleton**
   at PR #1754's merge time (sibling CD/EOCD rows use
   `other (CD/EOCD consistency)`). PR #1761 chose the same class
   for its corpus row, promoting the class from singleton to
   two-member. This bolsters the argument that the class is a
   legitimate factoring rather than a one-off — follow-up ZIP64-branch
   checks (PRs #1755/#1764) will likely push it to three+, matching
   the threshold paired-review #1754 §G.6 cited for skill promotion.
   ✓ (class-label divergence concern from §G.2 of predecessor
   review largely resolved by this PR's choice.)
