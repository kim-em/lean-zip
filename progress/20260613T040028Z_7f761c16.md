## Decisions / patterns

- **Avoid the range-fold for `set!` reasoning.** `blKraft` is a `List.range`
  fold; doing `set!`-increment proofs on it directly is painful. Adding a
  recursive `blKraftFrom` + a one-time bridge made every downstream step a
  short `omega` over atom-abstracted products. Same trick the existing
  `kraftSumFrom` machinery uses — worth the small duplication.
- **Additive `set!` lemmas, never subtractive.** Phrasing `blKraftFrom_set`
  as `lhs + old_term = rhs + new_term` sidesteps `Nat` truncated subtraction;
  `omega` then closes the corollaries given a single `(c−1)·w + w = c·w` fact.
- **`set`/`conv_rhs … from by omega` are Mathlib-isms** — not available here.
  Used `generalize hA : … = A at e1 ⊢` to abstract the nested `set!` arrays in
  `repairStep_kraft`, and `obtain ⟨c, hc⟩` + `rw` for the arithmetic rung.
- **Build/test on NixOS:** bare-shell `lake exe test` fails to *link*
  `libdeflate_ffi` (cached link flags / no zlib). Use
  `nix-shell --run "lake build -R && lake exe test"`. Pure-Lean module builds
  (`lake build -R Zip.Spec.HuffmanEncode`) work bare. The `-R` is needed after
  the worktree's cached config goes stale.
