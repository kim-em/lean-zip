## Key decisions / patterns

- **Reused `blKraft_set_dec`/`blKraft_set_inc2` at normalization level `b`** (not
  the real `maxBits`) for `repairStep_feas` region 3/2. They already absorb the
  nonlinear `c·2^w` algebra internally, so the prefix relations come out as clean
  linear facts and `omega` closes them.
- **`omega` + `2^k` positivity**: after omega lifts `2^(b-bits)` to `ℤ` it loses
  `≥ 0`, so inequality goals like `X ≤ X + 2^w` fail under `omega`. Used
  `Nat.le.intro` / `Nat.le_of_eq` on the raw Nat equation instead — robust.
- **`findBelow` well-definedness while over-subscribed**: peel-top scaling
  (`blKraftFrom_scale`) + `hfeas (maxBits−1)` gives `1 ≤ bl[maxBits]`; the
  `findBelow = 0` branch collapses `blKraft` to `bl[maxBits] ≤ blCountSum ≤
  2^maxBits`, contradicting over-subscription.
