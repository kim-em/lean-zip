# Progress: Track E FSE distribution decoder and table builder

**Date**: 2026-03-02T09:00 UTC
**Session**: fd4daa8c (feature)
**Issue**: #421

## Accomplished

- Implemented `Zip/Native/Fse.lean` (~140 lines):
  - `FseCell` / `FseTable` structures for decoding tables
  - `decodeFseDistribution`: reads accuracy log + normalized probabilities
    from a bitstream using RFC 8878 §4.1.1 variable-width bit scheme,
    with -1 (less-than-one) and repeat-zero support
  - `buildFseTable`: constructs decoding table from distribution using
    the position-stepping algorithm (step = tableSize/2 + tableSize/8 + 3),
    placing -1 symbols at table end, computing numBits/newState per cell
- Added `ZipTest/FseNative.lean` (~110 lines) with 8 tests:
  - Known distribution verification (symbol counts match probabilities)
  - Single-symbol and equal distribution edge cases
  - -1 probability placement at table end
  - Round-trip against FFI-compressed data
  - Stepping algorithm interleaving verification
  - numBits validation for low-probability symbols
- Registered module in `Zip.lean` and test in `ZipTest.lean`
- `lake build` and `lake exe test` pass, zero sorrys

## Key decisions

- Used `Int32` for probabilities to represent -1 (less-than-one) values
  per RFC spec, with `int32ToNat` helper for safe conversion
- FSE stepping algorithm only produces coprime steps for tableSize ≥ 32
  (accuracyLog ≥ 5), which matches the spec's minimum. Tests use valid
  accuracy logs accordingly.
- Kept implementation simple and readable per project conventions;
  no optimization passes yet
