## What was done

Converted all 12 `]!` runtime bounds in `Zip/Native/Gzip.lean` to

signatures, return types, and observable error strings are unchanged.

### Changes

* **`GzipDecode.decompress`** — wrapped the per-member loop body in
  `if hHdr : pos + 10 ≤ data.size then … else throw "Gzip: truncated header"`.
  All four header bytes (`id1`, `id2`, `cm`, `flg`) are now read **up front**
  inside the `then` branch, before any `unless`. This is necessary because
  do-notation reintroduces fresh names for `mut pos` after every `unless`,
  which drops the bound `hHdr` from scope. Reading first, branching second
  keeps the proof live for all four accesses.
* **`scanToZero` helper** — replaced the two
  `while pos < data.size && data[pos]! != 0 do pos := pos + 1`
  loops (FNAME, FCOMMENT) with a small well-founded recursive helper
  `scanToZero data pos`, terminating on `data.size - pos`. Same observable
  behaviour: returns the index of the next NUL or `data.size` if none.
* **`ZlibDecode.decompress`** — rewrote `if data.size < 6 then throw …` as
  `if hSz : data.size < 6 then throw … else <body>` so the `else` branch
  has `¬ data.size < 6` available. The `data[0]` and `data[1]` accesses are
  inside that else branch. Wrapped the Adler32 trailer reads with
  `if hT : pos + 4 ≤ data.size then … else throw "Zlib: truncated trailer"`,
  giving `hT` for `data[pos]`, `data[pos + 1]`, `data[pos + 2]`, `data[pos + 3]`.

### Patterns / lessons

* The do-notation `mut`-rebinding caveat from `proven-bounds` SKILL §7
  bites in a subtler form: not just after `if … then throw` (no else), but
  also after any `unless` whose body modifies control flow. Even when the
  `unless` body does **not** assign to `pos`, the elaborator threads
  `mut`-state through it and emits `pos : Nat` with no relation to the
  earlier `pos✝`. **Read every byte you need from the array first**, then
  do flow-control checks. This is worth adding to the skill — see Reflect.
* Lean's automatic `getElem` tactic discharges all the residual goals here
  (`pos < data.size`, `pos + 1 < data.size`, etc.) from `pos + 10 ≤ data.size`
  via `omega` automatically; no `'(by omega)` annotations were needed.
