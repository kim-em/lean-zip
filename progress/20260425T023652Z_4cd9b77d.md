## Construction note (sequencing)

The 6 override sub-checks at Zip/Archive.lean:396–413 fire in source
order (`cdSize` → `cdOffset` → `totalEntries` → `numberOfThisDisk` →
`diskWhereCDStarts` → `numEntriesThisDisk`). To pin the line-411
sub-check, every earlier sub-check must pass on its sentinel branch.
The construction:

| field                | std EOCD value | ZIP64 value | branch taken           |
|----------------------|----------------|-------------|------------------------|
| `cdSize`             | `0xFFFFFFFF`   | `55`        | sentinel pass          |
| `cdOffset`           | `0xFFFFFFFF`   | `45`        | sentinel pass          |
| `totalEntries`       | `0xFFFF`       | `1`         | sentinel pass          |
| `numberOfThisDisk`   | `0xFFFF`       | `0`         | sentinel pass          |
| `diskWhereCDStarts`  | `0xFFFF`       | `0`         | sentinel pass          |
| `numEntriesThisDisk` | `99`           | `1`         | **trip on line 411**   |

The planner-flagged risk in the issue body — that naive
`total_entries=99` would trip line 402 first — was avoided by
keeping `total_entries` at the default `SENTINEL_16`. The line-484
(now line 531) EOCD-internal consistency check is unreachable
because it lives downstream of the override branch in
`parseCentralDir`.
