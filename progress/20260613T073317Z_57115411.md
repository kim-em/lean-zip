### 3.2 — pre-sized packed-token array (landed)

- `lz77ChainIterP` / `lz77ChainLazyIterP` (the production matchers dispatched by
  `Zip/Native/DeflateDynamic.lean`) now seed their token accumulator with
  `Array.emptyWithCapacity data.size` instead of `#[]`, removing the geometric
  regrowth/copy of the token array. Token count is always ≤ `data.size`, so
  `data.size` is the exact (safe upper-bound) capacity hint.
- **Byte-identical**: `emptyWithCapacity n = #[]` by `rfl`
  (`Array.emptyWithCapacity_eq`, a `@[simp]` lemma). The packed→boxed
  equivalence proofs `lz77ChainIterP_eq` / `lz77ChainLazyIterP_eq` in
  `Zip/Spec/LZ77PackedCorrect.lean` carry through with that one lemma added to
  the seed-reduction `simpa only` set. No theorem statement changed.
- **Verified byte-identity by measurement**: a before/after `bench-report` over
  Canterbury shows `out_size` identical on every one of the 99 native rows
  (zero ratio drift — the acceptance criterion). Throughput delta is within
  this shared machine's run-to-run noise (median +0.1%, mean +0.16% across
  native rows; individual rows swing −7%…+8% from machine noise, dwarfing the
  issue's predicted ~1.4% token-build win). The change is a pure allocation
  reduction that cannot regress.

### 3.1 — UInt32 matcher tables (NOT pursued; retired on evidence)

The issue flags 3.1 as conditional ("pursue only if the proof spike closes
without ballooning"). It does not need a proof spike: it is **the same change as
the Wave-3a "UInt32 chain-state representation" already measured slower in
situ** and abandoned (`plans/track-d-state.md`, D-22 row + the Wave-3a verdict
block, lines 89–102): the production matcher's `hashTable`/`prev : Array Nat`
tables converted to `Array UInt32` measured *slower* (alice29 matcher ms: L1
6.1 vs 4.87, L6 23.7 vs 22.7), with full direct-contract proofs + a
token-identity gate preserved unmerged on `perf/3a-uint32-chain`. #2490's 3.1
text predates that verdict and also describes the older `Array Bool` validity
shape, which the production `lz77ChainIterP` (sentinel-`data.size` `Array Nat`)
already moved past. No actionable work remains, so I did not create a follow-up
issue — re-attempting would contradict measured evidence.
