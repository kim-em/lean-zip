## Accomplished

- Added the libdeflate-style hash3 singleton table (`hash3Single`,
  `hash3Probe`, `updateHash3` in `Zip/Native/Deflate.lean`): 15-bit
  3-byte-hash bucket holding the most recent position; probed once per
  position and verified (window guard + three byte compares) before use.
- Threaded the `h3tab` state through all six chain mainLoops
  (`lz77Chain`, `lz77ChainIter`, `lz77ChainLazy`, `lz77ChainLazyIter`,
  `lz77ChainIterP`, `lz77ChainLazyIterP`); the verified probe result seeds
  the chain walk, restoring length-3 matches lost when #2620 moved the
  chains to a 4-byte hash. The lazy lookahead walk deliberately skips the
  probe (deferral needs matchLen2 > matchLen ≥ 3, so a length-3 seed
  cannot change the outcome).
- Switched the L9/L10 candidate-cache build (`buildCache`,
  `insertChainRange` in `Zip/Native/DeflateParse.lean`) from the 3-byte
  chain hash (`hash3opt`, removed) to hash4 chains + the hash3 singleton,
  recording the verified length-3 candidate as cache slot 0 — unblocking
  hash4 at L9/L10 without the recorded x-ray/ooffice ratio regression.
- Re-threaded the spec proofs: `hash3Probe_spec` feeds `chainWalk_spec`'s
  initial-accumulator hypothesis; `chainWalk_fst_le` and the packed decode
  lemmas (`chainWalkGuardedPacked_mod`/`_div`) generalized from the 0-seed
  to any seed ≤ 511; lockstep equivalences re-threaded with `h3tab`.

## Key discovery

`Prod` projections of an `@[inline]` probe application inside a
well-founded mainLoop body send the **kernel** into deep recursion
("(kernel) deep recursion detected", all six matchers). Fix: pack the
probe result as `bestPos * 512 + bestLen` (the `chainWalkGuardedPacked`
idiom) and decode with `% 512` / `/ 512` at call sites. Minimal repro
confirmed projections → 15-min kernel blowup; packed → seconds.

  matcher half was reverted; implementation preserved at commit d4e86ca9.

- A merge-base worktree binary showed a +50% decode anomaly (cross-worktree
