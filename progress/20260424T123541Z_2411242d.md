# Track E: per-entry localOffset + 30 ≤ cdOffset archive-layout invariant

**Date**: 2026-04-24 UTC
**Session type**: feature
**Issue**: #1800
**PR**: #1813 (expected)

## Summary

`parseCentralDir` now rejects CD entries whose resolved `localOffset`
plus the 30-byte fixed LH header (APPNOTE §4.3.7) reaches into or past
the CD region. This is the per-entry micro-shape counterpart to the
archive-level macro-shape `cdOffset + cdSize ≤ eocdPos` guard (sibling
in-flight PR #1809). Together they characterize the legitimate archive
layout `[LH+data]* [CD] [EOCD]` at both granularities.

Pre-PR, `Archive.list` had no gate at all for this invariant — crafted
archives with `localOffset ≥ cdOffset` listed successfully. Only the
extract path's late LH-signature check (`"bad local header signature"`)
caught a subset of the construction, and it could be defeated by an
overlap where the CD bytes happened to match `sigLocal`.

## Changes

- **`Zip/Archive.lean`**: added a `cdFileOffset : Nat` parameter to
  `parseCentralDir` (threading the source-file position of the CD; the
  existing `cdOffset` parameter is buffer-relative and always `0` in
  practice). New check runs after ZIP64 resolution and before the
  stored-method size invariant, using the asymmetric
  `SpanInFile`-shaped subtraction
  `localOff ≤ cdFileOff ∧ 30 ≤ cdFileOff - localOff` to avoid `UInt64`
  wrap on crafted large `localOff`.
- **`scripts/build-cd-lh-mismatch.py`**: threaded a
  `local_hdr_offset` kwarg through `make_cd` / `write`. Default is `0`
  (load-bearing for all existing fixtures); only
  `cd-entry-localoffset-past-cdstart.zip` exercises the non-default
  branch.
- **`testdata/zip/malformed/cd-entry-localoffset-past-cdstart.zip`**:
  new fixture, 122 B, sha256
  `161f4b7a720cb4d5966bfa9613a6216315c4f3e289701c32e74d869f1fd5443a`.
  LH+data at file offset 0 (length 45), CD at offset 45,
  `localOffset=50` → fires new check (50+30=80 > 45) but would have
  passed the late span check (80 ≤ fileSize=122).
- **`ZipTest/ZipFixtures.lean`**: assertThrows on substring
  `"entry local offset overlaps central directory"`; added to cleanup
  loops.
- **`SECURITY_INVENTORY.md`**: *Recent wins* bullet + regression-table
  row. Also swept all CD-parse line references to account for the
  ~29-line shift introduced by the new guard (`sed`-style mapping:
  old line `L` → `L + 6` for L ≤ 571, `L + 29` for L > 571).
- **`.claude/skills/error-wording-catalogue/SKILL.md`**: new row for
  the substring.
