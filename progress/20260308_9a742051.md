## Accomplished

Proved two `decompressBlocksWF` theorems for the compressed block path
when `numSeq == 0` (literals only, no sequence commands):

1. **`decompressBlocksWF_single_compressed_literals_only`**: When a single
   last compressed block has numSeq=0, result is `output ++ literals` at
   `blockEnd = afterHdr + blockSize`.

2. **`decompressBlocksWF_compressed_literals_only_step`**: When a non-last
   compressed block has numSeq=0, it recurses with `output ++ literals`,
   updated Huffman table, unchanged FSE tables/offset history, and
   position at blockEnd.

## Key patterns discovered

- **Dependent match elaboration**: When a parameter `huffTree` appears in
  a hypothesis type (`hlit : ... = .ok (..., huffTree)`), Lean's match
  elaborator adds that hypothesis as an extra match discriminant in
  `if let some ht := huffTree then some ht else prevHuffTree`. Fix: use
  `id huffTree` to break the dependency chain.

- **Two-stage simp for Except.mapError**: In the compressed branch,
  `parseLiteralsSection` is wrapped in `.mapError`. Using `hlit'` directly
  in `simp only` causes proof-term contamination in match expressions.
  Fix: first `simp only` reduces bind/pure structure, then `rw [hlit']`
  substitutes cleanly, then second `simp only` finishes.

- **generalize + rw + exact heq**: For step theorems (non-last blocks),
  `generalize heq : <RHS> = rhs` before `unfold` prevents the RHS
  `decompressBlocksWF` call from being unfolded. After `simp + rw`,
  `exact heq` closes the goal.
