## Accomplished

- **`frameHeaderMinSize`** helper definition: computes minimum frame header
  size from the descriptor byte (~10 lines). Mirrors the field-size logic
  in `parseFrameHeader` (windowDescSize, didSize, fcsSize) to produce the
  exact number of bytes needed.

- **`parseFrameHeader_succeeds`** completeness theorem (sorry-free):
  proves that when data has the correct Zstd magic number and
  `data.size ≥ pos + frameHeaderMinSize data[pos + 4]!`,
  `parseFrameHeader` returns `.ok`.

## Proof approach

1. Case-split on result: `ok` case is trivial (`rfl`), `error` case
   derives `False`.
2. Single-pass simp reduces all monadic constructs in `hres`.
3. First 3 guards (magic size, magic value, descriptor size) cleared
   via `rw [if_neg ...]` using `omega` from `hmin` (all descriptors
   give `frameHeaderMinSize ≥ 6`, proved by `decide` over `Fin 256`).
4. `generalize` abstracts `ss`/`df`/`ff` (singleSegment, didFlag,
   fcsFlag) in BOTH `hres` and `hsize`, ensuring `split at hres`
   substitutes consistently.
5. `by_cases` on `ss` resolves the window descriptor branch.
6. `repeat (first | contradiction | (simp ... at hsize; omega) | (split at hres))`
   walks through all remaining if/match branches: `contradiction` closes
   ok-branches (`.ok _ = .error _`), `omega` closes error-branches
   (guard condition contradicts `hsize`).

## Key decisions

- Used `generalize` to synchronize case analysis between `hres` and
  `hsize`. Without this, `split at hres` creates branches where `hsize`
  still has abstract match expressions, making `omega` fail.
- Used `set_option maxRecDepth 4096` and `set_option maxHeartbeats 800000`
  for the deeply nested monadic expression.
- Suppressed `linter.unusedSimpArgs` because the `repeat` loop invokes
  the same `simp` call across goals that need different lemma subsets.
