## Decisions

- **Sentinel handling for ZIP64.** The standard EOCD uses `0xFFFF` in the
  16-bit `totalEntries` slot as the "consult ZIP64" marker. I preserved
  the existing `cdSize` / `cdOffset` override pattern — unconditional
  clobber when a valid ZIP64 EOCD64 is found, regardless of whether the
  16-bit field was `0xFFFF`. This keeps the three EOCD-metadata reads
  uniform and matches the spec's "ZIP64 wins when present" behaviour.
- **Gap tracking.** I added a *Missing work* bullet naming the two
  unaddressed CD-vs-EOCD dimensions (disk-number, comment-length) per
  the issue's explicit instruction. Those are non-goals of this PR but
  real gaps now that the CD-vs-EOCD category exists.
- **Fixture shape.** Both entries-on-disk (EOCD+8) and total-entries
  (EOCD+10) fields claim 2; the parser only reads `+10`, so the other
  is cosmetic but keeps the archive internally consistent from a
  producer's perspective. Using method=0 (stored) to keep the fixture
  byte-minimal and deterministic.
