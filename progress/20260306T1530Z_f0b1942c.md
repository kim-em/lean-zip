## Key decisions

- **Structural proof over evaluation**: The issue expected `decide_cbv` or
  `exact ⟨_, rfl⟩` to work after PR #765 (which replaced the `while` loop
  with WF recursion). However, `buildFseTable` also contains 5 `for` loops
  (`for ... in [:n]`) that use opaque `Std.Legacy.Range.forIn'` internals,
  preventing kernel evaluation. Instead of refactoring all loops to WF
  recursion (which would break existing proofs like `buildFseTable_cells_size`),
  we proved the function structurally cannot fail: each loop body only returns
  `.ok (.yield _)`, so forIn always returns `.ok`, and the bind chain always
  succeeds.

- **Stronger result**: `buildFseTable_ok` proves success for ALL inputs, not
  just the predefined distributions. This is stronger than the issue requested
  and useful for future specs.

## Patterns

- `forIn_range_always_ok` + `Except.bind_always_ok` pattern for proving do-blocks
  with for loops always succeed. Applicable to any Except monad function that
  uses for loops but never throws.
