## Decisions

- **Long-link symmetric fixture deferred.** The issue explicitly
  carved long-link out of scope ("one fixture exercises the shared
  guard and pins the long-name wording"). The 'K' arm guard is
  structurally identical to the 'L' arm and is caught by symmetric
  code review. This matches the project's existing
  `hardlink-outside.tar` positive-regression precedent.
- **Single-byte trailing NUL terminator in the fixture payload.** An
  attacker could plausibly supply a payload of arbitrary length and
  alignment; the chosen shape (13 meaningful bytes + 1 NUL
  terminator = `size 14`, padded to 512) is the minimal shape that
  still exercises the interior-NUL + `stripTrailingNuls` interaction.
- **Fixture lives in the existing Lean builder.** The sibling
  builders (`scripts/build-gnu-long-malformed-fixtures.lean`) already
  define the GNU long-name pseudo-header and trailing regular-entry
  helpers. Adding a fifth fixture keeps the builder cluster coherent
  and avoids a one-off Python script for a single 1.5 KB fixture.
