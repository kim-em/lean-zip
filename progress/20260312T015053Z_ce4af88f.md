- **Date**: 2026-03-12T01:50Z

- **Issue**: #1175

## Accomplished

Added two capstone theorems in `Zip/Spec/ZstdFrame.lean` composing the full
completeness chain from byte-level conditions to `decompressZstd` success:

1. **`decompressZstd_succeeds_single_raw_frame`** — byte-level frame header
   conditions + raw block conditions → `∃ output, decompressZstd data = .ok output`

2. **`decompressZstd_succeeds_single_rle_frame`** — same for RLE blocks

Each composes the chain:
- `parseFrameHeader_succeeds` (byte-level → frame header parsed)
- `decompressFrame_succeeds_single_raw/rle` (header + block → frame decompressed)
- `decompressZstd_single_frame` (frame + end-of-data → API success)

## Decisions

- **Universally quantified hypotheses**: Header field constraints and block
  conditions are quantified over `(hdr, afterHdr)` from `parseFrameHeader`,
  since the caller doesn't know these values without parsing. This follows
  the plan's suggested approach.

- **`hterm` hypothesis**: The frame-fills-data condition is stated as
  `∀ content pos', decompressFrame ... = .ok (content, pos') → pos' ≥ data.size`
  rather than byte-level, because there's no existing theorem characterizing
  `decompressFrame`'s exact output position. A future issue could add
  `decompressFrame_pos_eq_single_raw/rle` to make the chain fully byte-level.

- **`hframeSize` conversion**: The `parseFrameHeader_succeeds` type expects
  `data.size ≥ 0 + frameHeaderMinSize data[0 + 4]!` while the natural statement
  uses `data.size ≥ frameHeaderMinSize data[4]!`. Used `by simpa` to bridge.
