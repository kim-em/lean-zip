## What was done

Created `Zip/Spec/XxHash.lean` (~180 lines) with formal specification
predicates for the XXH64 hash function used by Zstd for content checksums.

### Specification predicates
- `ValidPrimes`: verifies all 5 XXH64 prime constants match the spec
- `ValidRound` / `RoundExpansion`: single accumulator round step
- `RoundState` / `initState`: 4-accumulator stripe processing state
- `ValidConvergence`: 4-accumulator merge after stripes
- `ValidAvalanche`: 3-step final mixing
- `ValidProcessRemaining`: tail byte processing (8/4/1-byte chunks)

All predicates have `Decidable` instances.

### Theorems proved (0 sorry)
- `primes_valid`: prime constants are correct (`decide`)
- `round_eq_expansion`: round = rotl(acc + lane*PRIME2, 31) * PRIME1 (`rfl`)
- `xxHash64_deterministic`: same input/seed → same output (`rfl`)
- `xxHash64Upper32_eq`: upper32 = (xxHash64 data 0 >>> 32).toUInt32 (`rfl`)
- `mergeAccumulator_eq`: expansion to XOR-round-multiply-add form (`rfl`)
- `avalanche_eq`: expansion to 3-step mixing form (`rfl`)

### Theorems stated (4 sorry)
- `xxHash64_empty`: empty input reduces to `avalanche (seed + PRIME5)`
- `empty_seed0`: test vector — empty input with seed 0
- `single_byte_0x42`: test vector — byte 0x42 with seed 0
- `upper32_empty`: test vector — upper 32 bits of empty hash

UInt64 kernel evaluation is too expensive for `decide_cbv` (heartbeat timeout).
Same test vectors are verified at runtime in `ZipTest/XxHashNative.lean`.
