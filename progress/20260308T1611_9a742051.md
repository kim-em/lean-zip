## Accomplished

Added two frame-level content characterization theorems to `Zip/Spec/Zstd.lean`:

1. **`decompressFrame_single_raw_content`**: When `decompressFrame` succeeds and the
   frame contains a single raw last block, the output equals the raw block content.

2. **`decompressFrame_single_rle_content`**: When `decompressFrame` succeeds and the
   frame contains a single RLE last block, the output equals the RLE block content.

Both theorems compose the block-level `decompressBlocksWF_single_raw`/`_single_rle`
theorems from #955 with frame-level unfolding (dictionary check, checksum validation,
content size validation).

## Proof structure

Each proof (~25 lines):
1. Derive `hoff` (block loop offset within bounds) from `parseBlockHeader_le_size`/`_pos_eq`
2. Compute exact block loop result via `decompressBlocksWF_single_raw`/`_single_rle`
3. Unfold `decompressFrame`, substitute `parseFrameHeader` result
4. Handle dictionary check via `split at hframe` (two cases: some dictId, none)
5. Unfold `decompressBlocks`, substitute known block result
6. Simplify `ByteArray.empty ++ block` via `ByteArray.empty_append`
7. Close with `grind` for remaining checksum/contentSize conditionals

## Decisions

- Placed theorems at end of file (before `end Zstd.Spec`) rather than after
  `decompressFrame_checksum_valid` as initially planned, because they depend on
  `decompressBlocksWF_single_raw`/`_single_rle` which appear later in the file.
- Named `_hdict` (underscore prefix) since the hypothesis is not directly used in
  the proof — `split at hframe` handles dictionary case splitting, and the `dictId != 0`
  case is dismissed by `nomatch`.
