## Decisions

- **Guard placement**: after the stored-method size invariant rather
  than before it. Both guards require post-ZIP64 resolved UInt64
  values; placing the new guard second keeps the stored-method message
  as the attribution for stored-method archives where both guards
  could fire, matching the issue's preferred ordering.
- **Same CRC on LH and CD**: the fixture uses `0xDEADBEEF` on both
  sides so the existing CD/LH `crc32` consistency check (PR #1728,
  `"crc32 mismatch between CD and local header"`) does not fire
  first. This pins attribution to the empty-entry invariant.
- **Empty payload**: `write(..., payload=b"")` because `compSize=0`
  means no bytes follow the LH fixed header — the 9-byte name bytes
  come immediately, then the CD. Without the `payload` kwarg the
  script appended the module-level `b"hello\n"` payload which would
  have desynced the CD offset from `cdOffset=39`.
- **Sentinel value `0xDEADBEEF`**: canonical "obviously crafted"
  nonzero UInt32; any nonzero value fires the same guard.
