## Key patterns / decisions

- **Decomposition of #1510**: the parent issue asked for all of `lz77Lazy`
  + `lz77LazyIter` together (~20 sites). Following cluster A's
  retrospective, that bundle caused `dsimp only` + `split` interactions
  at sites where the iter and recursive variants had to move in
  lockstep. Split into C1 (recursive, this PR) and C2 (iter, next
  session). C1 still needed a *minimal* iter change ‚Äî threading the
  new `hash3` proof argument ‚Äî but no iter-side dite refactoring.
- **Using `hlt` across the iter/recursive gap**: since
  `lz77LazyIter.mainLoop` directly calls `lz77Lazy.hash3`, both sides
  of `mainLoop_lazy_eq` produce the same `hash3 data pos hashSize hlt`
  term ‚Äî no `hash3_eq` bridge lemma was needed (unlike cluster A's
  greedy case, where the iter kept its own old-form `hash3`).
- **Literal bridge via `getElem!_pos`**: the remaining mismatches are
  only in the literal-emission branches, where iter still emits
  `.literal data[pos]!` but recursive now emits
  `.literal (data[pos]'h)`. One `getElem!_pos data pos (by omega)`
  rewrite per literal branch closes these goals.
