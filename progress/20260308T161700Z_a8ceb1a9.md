2. **`decompressBlocksWF_compressed_sequences_step`** — When the block loop

   history. Proof: `rw [eq_def]` (to unfold only LHS, not the recursive RHS) +

   mismatch in the Huffman table `if let` expression).

## Key patterns discovered

- **Step theorems need `rw [f.eq_def]` not `unfold f`**: `unfold` unfolds all
  occurrences including the recursive call on the RHS, creating an unresolvable
  mess. `rw [f.eq_def]` only rewrites the first match (different arguments
  prevent matching the RHS).
- **Dependent type gap in `if let` + simp**: When a hypothesis `hlit` mentions
  `huffTree` and the conclusion has `if let some ht := huffTree`, simp creates
  a dependent match `match huffTree, hlit with ...` on the RHS that doesn't
  match the LHS's simpler `match huffTree with ...`. Closed with
  `congr 1; cases huffTree <;> rfl`.
