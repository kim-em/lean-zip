## Key Techniques Learned
- `↓reduceIte` limitation: does NOT reduce `if (false = true)` (Prop from Bool equality).
  Use `if_neg (show ¬(false = true) from nofun)` instead.
- WF functions loop under `simp only [f]` — must use `unfold f` instead.
- `split at hspec` is the right pattern for handling `if` conditions in hypotheses
  from WF functions (replaces old `guard` + `by_cases` pattern).
