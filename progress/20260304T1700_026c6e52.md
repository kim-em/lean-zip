# Progress: FSE table resolution for sequence decoding

**Date**: 2026-03-04 17:00 UTC
**Session**: 026c6e52 (feature)
**Issue**: #473

## Accomplished

Implemented FSE table resolution — the connector between compression mode
parsing and sequence decoding for Zstd compressed blocks.

### Deliverables completed

1. **`buildRleFseTable`** — single-cell FSE table for RLE mode (accuracyLog=0).

2. **`resolveSingleFseTable`** — dispatches on `SequenceCompressionMode`:
   - Predefined (0): builds table from predefined distributions in `Fse.lean`
   - RLE (1): reads 1 byte, builds single-symbol table
   - FSE_Compressed (2): creates `BitReader`, decodes distribution, builds table
   - Repeat (3): returns error (needs cross-block state, future work)

3. **`resolveSequenceFseTables`** — resolves all three tables in RFC order:
   litLen (36 syms, maxAccLog=9), offset (32 syms, maxAccLog=8),
   matchLen (53 syms, maxAccLog=9).

4. **Tests** in `ZstdNative.lean`: RLE table construction, predefined/RLE/repeat
   single-table resolution, all-predefined and mixed-mode triple resolution.
