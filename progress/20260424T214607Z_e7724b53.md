## Summary

Closed the UStar layer of the smuggled-NUL attack class. `Tar.parseHeader`
now rejects an interior NUL byte (a NUL followed by any non-NUL byte) in
each of the three UStar string fields — `name` (offset 0, 100 B),
`linkname` (offset 157, 100 B), and `prefix` (offset 345, 155 B) — with
three distinct error substrings that keep failure attribution per-field.
The guards run after the checksum + magic checks (so header integrity is
confirmed first) and before any `Binary.readString` call, so a smuggled
`"evil.txt\x00.tar"` payload can no longer be silently truncated to
`"evil.txt"` (POSIX `open(2)` / `symlink(2)` truncation differential
versus `Tar.list` callers routing on `entry.path`).

## Decisions

- **Sibling-arm coverage policy.** Followed the PR #1865 long-link
  precedent: one fixture exercises one arm (`name`); the structurally
  identical `linkname` and `prefix` arms are covered by symmetric code
  review. The three throws share the same helper but distinct substrings
  so failure attribution still pins to the exact field.

- **Builder choice.** Created a new sibling builder
  (`build-ustar-malformed-fixtures.lean`) rather than extending
  `build-gnu-long-malformed-fixtures.lean`. The two scripts target
  different header types (GNU pseudo-entries vs. plain UStar regular-file
  headers), so a separate file keeps the `main` dispatch obvious for
  future contributors.

- **Comment block.** Inline comment at the new throws cites all three
  sibling guards (PR #1831 ZIP CD, PR #1865 GNU long-name/long-link,
  PR #1866 PAX silent-skip) with their error substrings, plus the
  writer-side `Binary.writeString` NUL-padding-only invariant — so
  someone reading the parser code sees the full attack-class context and
  the writer-side guarantee that legitimate archives never trip the
  guard.
