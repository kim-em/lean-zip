## Decisions and patterns

- **Why a helper (`freqsToPairs`) instead of inlining `.pmap`**: the
  first two attempts inlined either `(List.range n).attach.map ...` or
  `(List.range n).pmap ...` directly into `deflateDynamic`. Both
  triggered `maximum recursion depth has been reached` on the `rfl`
  calls inside `deflateDynamic_spec.hdef` (the proof that
  `deflateDynamic data = bw_flushed`). Hiding the `pmap` call behind a
  function symbol (`freqsToPairs`) lets `rfl` succeed without the
  elaborator recursing into the `pmap` definition on both sides.
  `List.pmap` is `@[expose]`, so inline uses are eagerly reduced,
  which is what was hitting the limit.

- `freqPairs_witness` was rewritten to return the `(sym, freqs[sym]'hsym)`
  witness rather than `(sym, freqs[sym]!)`. The `hfreq : freqs[sym]! ≥ 1`
  hypothesis is bridged via `getElem!_pos` since `sym < freqs.size`.
