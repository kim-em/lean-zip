# Meditate: bare simp campaign retrospective and workflow pattern extraction

**Date**: 2026-03-04 09:00 UTC
**Session**: 38f1595e (meditate)
**Issue**: #541

## What was done

Third and final meditate in the bare simp campaign arc (#381 → #477 → #541).
Updated 2 existing skills and created 1 new skill based on patterns from
20 progress entries and 17+ merged PRs since the last meditate.

### 1. proof-review-checklist — batch workflow and stale-count verification

- Added **Phase 0** (verify issue accuracy): workers now verify actual bare simp
  counts on master before starting, catching stale issues where other PRs already
  cleaned the file. Includes the accurate grep command that excludes `simp only`,
  `simp_all`, etc.
- Replaced the "do not batch" guidance with **batch discovery, sequential application**:
  convert all bare simps to `simp?` simultaneously, build once to collect all
  suggestions, then apply in small batches. This was the key efficiency discovery
  from session 7f94ec69.

### 2. lean-simp-tactics — three new patterns

- **One-step nomatch**: `exact nomatch (h ▸ hyp)` for substitution+discrimination
  in a single step (cleaner than the existing two-step pattern).
- **`let` binding unfolding**: `simp only [binding_name]` unfolds local `let`
  bindings — useful when bare simp was doing unfold + simplify.
- **Lemma name guessing warning**: Table of common mistakes (`not_lt` vs
  `Nat.not_lt`, `beq_iff_eq` vs `beq_eq_false_iff_ne`). Always use `simp?`
  discovery, never guess.

### 3. agent-pr-recovery — new skill

Created a new skill for recovering agent PRs with merge conflicts, covering:
- **Cherry-pick rebase pattern**: the standard recovery workflow (identify content
  commits, cherry-pick to fresh branch from master, verify build)
- **Salvage vs. redo decision tree**: when to cherry-pick vs. start over vs. close
- **Stale issue detection**: how to check if a PR's work was already done
- **Rapid merge cadence mitigation**: short-lived branches, targeted builds,
  rebase-before-push

## Decisions

- Kept the existing `lean-simp-tactics` resistant patterns sections unchanged —
  they're comprehensive and accurate. Only added new content.
- Named the new skill `agent-pr-recovery` rather than `agent-coordination` to
  be specific about its trigger conditions (PR recovery, not general coordination).
- Did not modify agent-worker-flow — it covers general workflow, the new skill
  covers the specific recovery patterns.
