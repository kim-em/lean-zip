## (b) Dead-branch pattern consistency

Three distinct else-branch shapes, all deliberate:

1. **Outer `hht`/`hhv` else** (no valid hash index yet) тАФ
   emit literal, advance by 1. **Byte-identical** between #1517
   (`lz77GreedyIter.mainLoop`) and #1529 (`lz77LazyIter.mainLoop`):
   ```lean
   mainLoop data windowSize hashSize hashTable hashValid (pos + 1)
     (acc.push (.literal (data[pos]'(by omega))))
   ```

2. **Inner `hht2`/`hhv2` else** (#1529 only, lazy lookahead) тАФ
   commits the match already found at `pos` instead of a literal.
   Logically correct: the lookahead is optional, so a failed lookup
   means "don't bother", not "no match".

3. **`emitTokensWithCodes` else** (#1525) тАФ recurses without emitting.
   Dead by `nativeFindLengthCode_idx_bound` + `hlit_size`; survives
   only for the elaborator.

Pattern (1) is a candidate for a shared helper once `mainLoop` is
hoisted out of its `where` block тАФ but that's a larger refactor, not a
one-lemma cleanup.
