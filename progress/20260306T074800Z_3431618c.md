# Progress: resolveOffset exact return values for repeat codes

- **Date**: 2026-03-06T07:48:00Z
- **Session type**: feature
- **Issue**: #674

## Accomplished

Added three characterizing theorems to `Zip/Spec/ZstdSequence.lean` that
specify the exact return value of `resolveOffset` for repeat offset codes
1–3 when `literalLength > 0`:

- `resolveOffset_repeat1_val`: rawOffset=1 → `history[0]!`
- `resolveOffset_repeat2_val`: rawOffset=2 → `history[1]!`
- `resolveOffset_repeat3_val`: rawOffset=3 → `history[2]!`

These are **characterizing properties** (level 2 specs) — they state which
history entry the RFC 8878 repeat offset mechanism returns, complementing
the existing positivity theorems that only proved the result is > 0.

## Proof technique

All three proofs follow the same one-line pattern used by the existing
`resolveOffset_positive_litLen_pos`: `simp only [resolveOffset, ...]` with
explicit `show` terms for the numeric guards. The implementation unfolds
cleanly because `rawOffset` is a concrete literal in each theorem.
