# Progress: parseSequencesHeader numSeq characterization

**Date**: 2026-03-07 UTC
**Session**: feature (6626fe38)
**Issue**: #806

## Accomplished

Added two sorry-free theorems to `Zip/Spec/ZstdSequence.lean`:

1. **`parseSequencesHeader_numSeq_small`**: When 0 < byte0 < 128,
   numSeq = byte0. This is the 1-byte encoding extraction for the most
   common case (up to 127 sequences per block).

2. **`parseSequencesHeader_numSeq_zero_iff`**: For byte0 < 128,
   numSeq = 0 ↔ byte0 = 0. Complete zero/positive characterization
   for the 1-byte encoding.

## Proof technique

Used `by_cases` + `rw [if_pos/if_neg]` for monadic unfolding through
the `parseSequencesHeader` do-block guards, rather than `split at h`
which has trouble with inaccessible hypotheses from Bool `BEq` conditions
and guard-generated `Except.bind` wrappers.

The `numSeq_zero_iff` forward direction reuses `numSeq_small` to derive
contradiction when byte0 > 0 but numSeq = 0.
