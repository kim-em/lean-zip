## Patterns / decisions

- The issue body suggested adding a `gnameOverride : Option String`
  field on the builder helper-data struct, citing parallels to
  PR #1937's `pathOverride`, PR #1934's `linknameOverride`, and
  PR #1944's `unameOverride`. Only `pathOverride` actually exists —
  `path` is auto-split via `splitPath` inside `buildHeader`, so an
  override is needed to inject a NUL through that branch.
  `linkname` / `uname` / `gname` are written verbatim by
  `Binary.writeString`, so the existing entry-struct fields work
  directly without an `Option String` indirection. Followed the
  PR #1934 / #1944 actual pattern (direct field assignment) rather
  than the issue body's misremembered parenthetical.
