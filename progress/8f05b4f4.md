### Bug fixes
1. **Huffman direct/FSE condition swap** (ZstdHuffman.lean):
   `parseHuffmanTreeDescriptor` had headerByte >= 128 dispatching to FSE instead
   of direct mode. Swapped to match reference zstd implementation.

2. **FSE two-state interleaved decoding** (Fse.lean):
   `decodeFseSymbolsAll` now initializes two states and alternates between them
   per `FSE_decompress1X`, with overflow-based termination.

3. **NCount readProbValue bit combination** (Fse.lean):
   The variable-length probability decoder combined `rawBits * 2 + extraBit`,
   placing first-read bits in the HIGH position. In LSB-first reading, first-read
   bits are the LOW bits. Also applied threshold adjustment unconditionally instead
   of only when the high bit is set. Fixed to match `FSE_readNCount_body`.

4. **buildFseTable simplified** (Fse.lean):
   Replaced complex doubleCells/highBit formula with simpler reference formula:
   `nextState = count + stateIdx`, `nbBits = accLog - log2(nextState)`,
   `newState = (nextState << nbBits) - tableSize`.

5. **BackwardBitReader improvements** (Fse.lean):
   Added `startPos` boundary field and `totalBitsRemaining` for overflow detection.
   Fixed `readBits` loop to respect `startPos` boundary.

6. **decodeHuffmanSymbol end-of-stream** (ZstdHuffman.lean):
   Handles case where fewer than `maxBits` remain by left-aligning the available
   bits for table lookup.
