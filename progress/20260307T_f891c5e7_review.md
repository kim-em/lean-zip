## Key patterns discovered

- `iterate N (all_goals (try split at h))` is more reliable than `repeat split at h`
  for decomposing deeply nested monadic guard chains, because `repeat` can exit early
  when it can't find more to split on a single goal, while `iterate` + `all_goals`
  processes all branches in each pass
- `Decidable.not_not` is available without Mathlib (unlike bare `not_not`)
- For `bne` guard decomposition: `bne_iff_ne` + `ne_eq` + `Decidable.not_not` converts
  `¬(a != b) = true` to `a = b`
