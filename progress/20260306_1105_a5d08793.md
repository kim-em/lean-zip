## Accomplished

Proved content theorems for `copyBytes` and `copyMatch` that specify what bytes
are written at new positions (≥ original buffer size):

1. **`copyBytes_getElem_ge`**: For `j < count` with `srcPos + count ≤ src.size`,
   `(copyBytes dst src srcPos count)[dst.size + j]! = src[srcPos + j]!`.
   Proof by induction on `count`, using `push_getElem!_eq` for the base byte and
   the IH for subsequent bytes.

2. **`copyMatch_getElem_ge_nonoverlap`**: When `offset ≥ length` and `offset ≤ buf.size`,
   for `j < length`, `(copyMatch buf offset length)[buf.size + j]! = buf[buf.size - offset + j]!`.
   Proof via a loop invariant `copyMatch_loop_getElem_ge_nonoverlap` that threads
   the prefix preservation property through the induction.

3. **`push_getElem!_eq`** helper in `ZipForStd/ByteArray.lean`:
   `(buf.push b)[buf.size]! = b`.

## Key decisions

- The `copyMatch_getElem_ge_nonoverlap` hypothesis `srcPos + count ≤ src.size` was
  chosen over `srcPos + j < src.size` to give a cleaner induction. The former
  implies the latter for all `j < count`.
- The loop invariant for `copyMatch` threads `hprefix : ∀ i, i < buf.size → b[i]! = buf[i]!`
  through the induction, which is necessary to show that source indices (within the
  original buffer) still hold their original values as the buffer grows.
