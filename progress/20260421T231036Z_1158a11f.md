# 2026-04-21T23:10Z — feature 1158a11f — Track E symlink/hardlink policy + fixtures (#1549)

## Issue

#1549 — Track E Priority 1: document tar symlink and hardlink
extraction policy explicitly and add archive-slip regression fixtures.

## What was done

- Added a `typeHardlink : UInt8 := 0x31` named constant in
  `Zip/Tar.lean` alongside the existing `typeSymlink` etc., and
  referenced it in the `extract` fallthrough comment so the
  silent-skip policy is visible to readers.
- Rewrote the `Tar.extract` docstring (was stale: claimed symlinks were
  skipped, even though they have been validated and created since
  earlier work) to enumerate the per-typeflag policy: regular and
  directory entries go through `Binary.isPathSafe`; symlink linknames
  are component-checked for absolute, backslash, and `..`; everything
  else (hardlink, devices, FIFO, GNU sparse) silently skips its
  payload.
- Added `scripts/build-symlink-hardlink-malformed-fixtures.lean`
  (modelled on `build-pax-malformed-fixtures.lean`). It emits two
  byte-deterministic single-block UStar archives:
    * `testdata/tar/security/symlink-absolute.tar` — typeflag '2',
      linkname `/etc/passwd`.
    * `testdata/tar/security/hardlink-outside.tar` — typeflag '1',
      linkname `../outside`.
- Extended `ZipTest/TarFixtures.lean` with three new asserts:
    * `symlink-absolute.tar` → `"unsafe symlink"` (rejection happens
      before any `Handle.createSymlink` call).
    * `backslash-slip.tar` → `"unsafe path"`. The orphaned fixture is
      a regular-file entry (`typeflag == '0'`), so `Binary.isPathSafe`
      rejects it on the backslash before the symlink branch can fire.
    * `hardlink-outside.tar` → extraction succeeds with an empty
      extract directory. This guards against accidental promotion to
      a `createHardlink` call.
  Cleanup loops updated for the three new tmp paths.
- Added a "Symlink/hardlink extraction policy" subsection to
  `SECURITY_INVENTORY.md` § "Tar Parser/Extractor" and removed the
  now-stale "document and test symlink-handling policy explicitly"
  bullet.
- Ticked the Priority 1 bullet in
  `plans/track-e-current-audit-checklist.md`, citing the new fixture
  filenames.

## Decisions

- Where to place the `paddingFor 0` archive: I omitted trailing
  zero-blocks since `Tar.forEntries` already terminates on a short
  read (`readExact input 512` → `block.size < 512` ⇒ break). This
  matches the `tar-slip.tar` / `symlink-slip.tar` precedent and keeps
  the fixtures at exactly 512 bytes.
- For the hardlink test, I dropped a paranoia "no `/tmp/outside` in
  the parent" check because it is brittle: another process could
  create that file. The empty-extract-dir check is the precise
  invariant we want.
- Verified determinism by running the builder twice and `diff`-ing
  the outputs. SHA-256s recorded:
    * `symlink-absolute.tar` —
      `a6084c32aad5ddbe8072ff50ea48c1042af4150f7524b14024d865ff261be57e`
    * `hardlink-outside.tar` —
      `3fa9dd3e98ce883617dd0c5683c7707f157cc576ce938d8907f481ef25cdb921`
