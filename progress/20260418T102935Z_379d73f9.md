# 2026-04-18 10:29 UTC — feature — #1523 proven-bounds in lz77LazyIter (C2)

## Summary

Converted the 7 `]!` runtime bounds checks in `lz77LazyIter.trailing`
and `lz77LazyIter.mainLoop` to statically-proven bounds accesses and
repaired the spec proof cascade in `Zip/Spec/DeflateFixedCorrect.lean`.

- `lz77LazyIter.trailing`: `if h : pos < data.size` guard + `data[pos]'h`.
- `lz77LazyIter.mainLoop`: added `if hht : hsh < hashTable.size then` /
  `if hhv : hsh < hashValid.size then` guards for the outer hash-table
  reads, and analogous `hht2`/`hhv2` guards for the pos+1 peek.
  Fallback branches emit a reference-at-pos (or literal) and continue.
  All 5 `data[pos]!` call-sites replaced with `(data[pos]'(by omega))`.

## Spec repair

- `trailing_lazy_eq`: dropped the `getElem!_pos data pos hlt` bridge
  (both sides now use `data[pos]`).
- Added `lz77Lazy_updateHashes_fst_size` and `_snd_size` size-preservation
  lemmas mirroring the greedy versions.
- `mainLoop_lazy_eq` gained `hht : hashTable.size = hashSize`,
  `hhv : hashValid.size = hashSize`, `hhs : 0 < hashSize` parameters.
  These are needed to reduce the two outer and two inner `if h : i < _.size`
  dites via `Nat.mod_lt` on `lz77Lazy.hash3`.
- The IH is invoked with sizes re-established through
  `(lz77Lazy_updateHashes_fst_size ..).trans hht'` after `updateHashes`
  and directly through `hht'`/`hhv'` when only `set!` was applied.
- `lz77LazyIter_eq_lz77Lazy` supplies the new arguments via
  `(by simp) (by simp) (by decide)` — the `replicate 65536` ensures the
  sizes match at the top-level call.
