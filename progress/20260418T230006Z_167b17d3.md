## Deliverables completed

1. **`scripts/build-gnu-long-malformed-fixtures.lean`** — new builder
   script. Uses `Tar.buildHeader`, `Tar.paddingFor`, and `Binary.zeros`
   to emit byte-deterministic fixtures.
2. **Four new fixtures under `testdata/tar/malformed/`**:
   - `gnu-longname-truncated.tar` (612 B) — typeflag `'L'`, declares
     `size = 200` but stream ends after 100 bytes of payload. Triggers
     `tar: unexpected end of archive reading entry data`.
   - `gnu-longname-no-terminator.tar` (1536 B) — typeflag `'L'`,
     payload is exactly 100 bytes of `0x61` ('a') with no trailing
     NUL. `stripTrailingNuls` returns the full 100 bytes;
     `String.fromUTF8?` succeeds; the trailing regular-file entry
     inherits the 100-character path.
   - `gnu-longname-invalid-utf8.tar` (1536 B) — typeflag `'L'`,
     payload `0xFF 0xFF 0xFF 0x00`. After NUL stripping,
     `String.fromUTF8?` returns `none` and `Binary.fromLatin1`
     produces the expected three-codepoint string of U+00FF.
   - `gnu-longlink-truncated.tar` (612 B) — typeflag `'K'`, structurally
     identical to the long-name truncated fixture; ensures the long-link
     arm of `forEntries` stays in lock-step with the long-name arm.
3. **`ZipTest/TarFixtures.lean`** — added one `assertThrows` block
   covering both truncated fixtures and one non-throwing block
   asserting the post-decode path for the no-terminator and
   invalid-UTF-8 fixtures. Cleanup loop extended with the four new
   temp-file names.
4. **`plans/track-e-current-audit-checklist.md`** — moved the GNU
   long-name bullet from `- [ ]` to `- [x]` with a parenthetical
   listing the four fixtures.
