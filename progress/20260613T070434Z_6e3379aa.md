## Correctness / spec-taxonomy audit (Deliverable 2)

Verdict: the `lz77Optimal_*` theorems are genuine **characterizing**
properties (CLAUDE.md taxonomy level 2), not tautological restatements.

- `lz77Optimal_valid` : `ValidDecomp data 0 (lz77Optimal data).toList`.
  `ValidDecomp` is an independently-defined inductive (literals carry the
  correct byte; references satisfy `len≥3`, `1≤dist≤pos`, in-window
  byte-match; tokens cover `data[pos..]` contiguously). Characterizing. ✓
- `lz77Optimal_resolves` : resolving the produced token stream through the
  **RFC reference resolver** `Deflate.Spec.resolveLZ77` recovers
  `data.data.toList`. Gold-standard roundtrip/invertibility against a
  resolver that is entirely independent of the parser. Strong. ✓
- `lz77Optimal_encodable` : every emitted token meets the DEFLATE encoder
  bounds (3≤len≤258, 1≤dist≤32768). Genuine encoder-contract property. ✓
- `lz77OptimalIter_eq_lz77Optimal` : ties the runtime entry point to the
  reference recursive version. **Real, not vacuous** — proven via
  `optimalEmitIter_eq_emit` (strong induction: iter = `acc ++ rec.toArray`),
  and both `lz77Optimal`/`lz77OptimalIter` feed the *same* `computeChoices
  data`, so the choice arrays coincide. The iter-side contracts
  (`lz77OptimalIter_valid/_resolves/_encodable`) are derived from this
  equivalence and are exactly what the encoder pipeline consumes:
  `DeflateDynamic.lean:948` calls `lz77OptimalIter data`, and
  `DeflateBlockSplit.lean:1246-1247` consume `lz77OptimalIter_encodable` /
  `_resolves`. The verified path is the shipped path. ✓

Design strength worth recording: every theorem is stated for **arbitrary**
`chLen`/`chDist` choice arrays — nothing about the heuristic DP enters the
proofs, because `optimalEmit` re-verifies each match (guard + fresh
`countMatch`) and falls back to a literal otherwise. The proofs are therefore
immune to future cost-model tuning. No spec weakness found; optimality is
(correctly, per spec-quality discipline) measured not certified.
