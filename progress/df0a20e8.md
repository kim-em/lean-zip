### Proof fixes (7 files)
The `ite` → `dite` and `getElem!` → `getElem` changes required updates
across all proof files that reason about these functions:

- **BitReaderInvariant.lean**: Added `split at h` for codeLengthOrder dite
- **DecodeCorrect.lean**: Bridge lemmas via `getElem!_pos`, reverse-rewrite
  strategy for getElem matching
- **DecodeComplete.lean**: `erw` for getElem matching, `↓reduceDIte`
- **DynamicTreesCorrect.lean**: `split at h` + `getElem!_pos` bridge
- **DynamicTreesComplete.lean**: Added `hncl` parameter for bounds
- **InflateRawSuffix.lean**: `dif_neg` for dite, `erw` + `getElem!_pos`
  bridge for lengthExtra/distExtra

### Key proof technique
`getElem` and `getElem!` are different functions — `isDefEq` cannot
bridge them. Use `getElem!_pos` to convert between them, and `erw`
(not `rw`) to match getElem terms with different proofs.
