## Adversarial check

Per the issue's verification step, temporarily wrapped the `else`
body in `Zip/Tar.lean`'s `Tar.extract` with:

```lean
if e.typeflag == typeHardlink || e.typeflag == 0x36 then
  skipEntryData input e.size
else
  throw (IO.userError s!"adversarial: unexpected typeflag {e.typeflag.toNat}")
```

Result: `hardlink-outside.tar` and `tar-fifo-skipped.tar` continued
to pass; `tar-chardev-skipped.tar` fired with
`uncaught exception: adversarial: unexpected typeflag 51` (where
`51 = 0x33 = '3'`). Confirms the new test arm independently fires
on a regression to the chardev arm even with the hardlink and FIFO
arms preserved. The `Zip/Tar.lean` change has been reverted in the
worktree (`git diff Zip/Tar.lean` is clean).
