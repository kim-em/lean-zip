### Limitation: `partial` while loop

The `while` keyword in Lean 4 `do` notation uses `Lean.Loop.forIn`, which is
defined as `partial` — the kernel treats it as an axiom. This makes it
impossible to prove `weightsToMaxBits = weightsToMaxBitsSpec` without `sorry`.
The bridge lemma `weightsToMaxBits_eq_spec` documents this limitation.

The mathematical content is fully proven in `weightsToMaxBitsSpec_valid`.
To eliminate the bridge sorry, `weightsToMaxBits` would need to be refactored
to use explicit recursion instead of `while`.
