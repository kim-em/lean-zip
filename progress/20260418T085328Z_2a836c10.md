## Key patterns / decisions

- **Extra size guards in `mainLoop`**: to avoid proving updateHashes
  size preservation inline (which is awkward with `where`-block helpers),
  the iter `mainLoop` now has `if hht : hsh < hashTable.size then`
  guards. The else branch emits a literal, which is extensionally
  equivalent to the original `!`-access (which returned `default = 0`
  with `isValid = false`, also falling through to a literal).
- **`conv => lhs; rw` vs `conv_lhs`**: `conv_lhs` is not available in
  Lean 4.29.1 core; `conv => lhs; rw [...]` works.
- **Dependent-motive issues in rewriting**: when a term appears both
  as an index and inside a dependent `if h : P` condition, `rw` fails
  with "motive not type correct". `simp only` handles this better,
  so the final proof uses `simp only [..., ← hash_eq, hget_t, hget_v]`
  instead of sequential `rw`s.
- **`Bool.eq_true_of_ne_false` is not in core**: used
  `Bool.eq_false_iff.mpr` instead to turn `≠ true` into `= false`.
