# 20260424T130406Z — feature session b0530a2f

Closes #1811: Track E CD-parse hardening. Reject CD entries whose
`internalFileAttributes` field (APPNOTE §4.4.10, CD offset +36,
UInt16) has any reserved bit set (bits 1-15). APPNOTE defines only
bit 0 ("apparent ASCII/text data"); bits 1-2 are reserved for
PKWARE and bits 3-15 are unused in version 1.0.

## What shipped

- `parseCentralDir` in `Zip/Archive.lean` now reads `internalAttrs`
  at `pos + 36` immediately after the `diskNumberStart` check and
  rejects with `"CD entry internalAttrs reserved bits set"` unless
  `internalAttrs &&& 0xFFFE == 0`. Preserves Info-ZIP bit-0
  text-flag interop (verified against all 6 fixtures in
  `testdata/zip/interop/`: every CD entry uses `0x0000` or
  `0x0001`).
- New fixture `testdata/zip/malformed/cd-entry-internal-attrs-reserved.zip`
  (122 B, SHA-256 `de61dc3b08261e1b8ad38e30851a7d5aca1acac557838d2949625b13e464fb51`).
  Built deterministically by extending
  `scripts/build-cd-lh-mismatch.py`: new `internal_attrs` kwarg
  on `make_cd` (default `0` — byte-identity preserved for all
  existing fixtures, verified via sha256 compare before/after).
- `ZipTest/ZipFixtures.lean` gains an `assertThrows` block slotted
  between `cd-entry-disknum-mismatch.zip` and `cd-stored-size-mismatch.zip`
  matching `"internalAttrs reserved bits set"`.
- `error-wording-catalogue` skill updated with the new substring row.
- `SECURITY_INVENTORY.md` line anchors shifted by +20 from old
  Archive.lean line ≥ 502 (the insertion spans 20 lines: 17-line
  comment + 3-line read/check). `scripts/check-inventory-links.sh`:
  `errors=0, warnings=0`.
