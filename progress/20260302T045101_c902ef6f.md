## Accomplished

- Implemented `parseLiteralsSection` (RFC 8878 §3.1.1.3.1) in `Zip/Native/ZstdFrame.lean`:
  - Raw literals: variable-width header (1-3 bytes, 5/12/20-bit sizes), verbatim copy
  - RLE literals: variable-width header + 1 byte replicated `regeneratedSize` times
  - Compressed/treeless: descriptive error messages until Huffman infrastructure exists

- Implemented `parseSequencesHeader` (RFC 8878 §3.1.1.3.2):
  - All three sequence count formats (1/2/3 byte)
  - Handles 0-sequence case (no compression modes byte)
  - Includes compression modes byte in position for non-zero sequences

- Wired both functions into `decompressBlocks` compressed block handler:
  - Compressed blocks with 0 sequences and raw/RLE literals fully decompress
  - Blocks with sequences return specific error with sequence count

- Added 12 new tests (28-39) covering:
  - Raw/RLE literal parsing with 1-byte and 2-byte headers
  - Compressed/treeless literal rejection with clear errors
  - All sequence count formats (0, small, 2-byte, 3-byte)
  - Truncation error handling
  - FFI-compressed block header parsing verification

- Updated existing tests to accept new error messages from deeper parsing
